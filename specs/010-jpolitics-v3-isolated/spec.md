# Feature Specification: 정치쇼츠 V3 — @김정치입니다 격리 모드

**Feature Branch**: `010-jpolitics-v3-isolated`
**Created**: 2026-06-05
**Status**: Draft
**Input**: User description: "정치쇼츠 V3 — @김정치입니다 격리 모드"

## 개요

YouTube 채널 [@김정치입니다](https://www.youtube.com/@김정치입니다)의 영상 제작 방식(고정 노란 헤드라인, 4종 레이아웃, 인물 카드, 정당 컬러)을 자동으로 재현하는 정치 쇼츠 생성기를 신설한다. 기존 V1(`political`)·V2(`political_pro`) 파이프라인과 **물리적으로 완전 격리**되며, 메인 페이지의 진입 버튼 1개만이 유일한 접점이다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 단일 인물 인터뷰 영상으로 정치 쇼츠 만들기 (Priority: P1)

크리에이터가 정치인 1인의 인터뷰/연설 YouTube 영상 URL을 입력하면, 시스템은 자동으로 자막을 추출하고 영상의 핵심 발언을 분석하여 60초 이내 쇼츠로 변환한다. 결과 영상은 상단에 고정 노란 헤드라인이 떠 있고, 화면 중앙엔 인물 원본 클립이 나오며, 하단엔 실시간 자막이 표시된다.

**Why this priority**: 채널 콘텐츠의 가장 흔한 포맷(Talking Head)이며, 4종 레이아웃 중 가장 단순하고 다른 레이아웃의 기반이 되기 때문이다. 이 P1만 완성해도 채널 영상의 약 50%를 재현할 수 있다.

**Independent Test**: 조국 사퇴 인터뷰 영상(샘플 S1) URL 입력 → 60초 쇼츠 1편 출력 → 상단 헤드라인, 하단 자막, 출처 라벨 3요소 시각 확인.

**Acceptance Scenarios**:

1. **Given** 정치인 인터뷰 YouTube URL이 주어졌을 때, **When** 사용자가 V3 진입 버튼을 누르고 URL을 입력하면, **Then** 시스템은 3개의 기획안을 5분 이내에 제시한다.
2. **Given** 사용자가 기획안 중 하나를 선택했을 때, **When** "영상 만들기"를 누르면, **Then** 시스템은 30~60초 길이의 9:16 세로 영상 1편을 7분 이내에 생성한다.
3. **Given** 생성된 영상이 있을 때, **When** 재생하면, **Then** 모든 씬에 상단 노란 헤드라인이 고정되어 있고, 하단에 실시간 자막이 표시되며, 외부 인용 시 하단 출처 라벨이 보인다.

### User Story 2 — 두 정치인의 발언 대결을 VS 카드로 시각화 (Priority: P2)

크리에이터가 두 정치인이 서로 다른 입장으로 대립하는 사건의 영상이나 주제를 입력하면, 시스템은 도입부 씬에 **좌·우 VS 분할 카드**(각 인물 사진 + 정당 컬러 배경 + 이름)를 자동 구성한 뒤, 본편에서 발언 클립을 교차로 보여준다.

**Why this priority**: 채널의 두 번째 인기 포맷이며, 정치 콘텐츠의 핵심인 "대립 구도"를 시각화하는 차별점이다. P1 위에 인물 카드 페치 + VS 레이아웃만 추가하면 완성된다.

**Independent Test**: "양향자 vs 추미애 경기도지사 대결" 주제 입력 → VS 카드 도입부 + Talking Head 본편 영상 1편 출력 → 좌(민주당 파랑) / 우(국힘 빨강) 정당 컬러와 인물 사진이 정확한지 확인.

**Acceptance Scenarios**:

1. **Given** "A 후보 vs B 후보" 형태 주제가 입력되면, **When** 시스템이 기획안을 생성하면, **Then** 도입부 씬의 레이아웃이 `vs_card`로 자동 분류된다.
2. **Given** 두 인물 이름이 식별되면, **When** 인물 카드를 페치하면, **Then** 시스템은 각자의 정면 사진과 소속 정당 컬러를 자동 매칭하여 좌·우에 배치한다.
3. **Given** 두 인물 중 한 명의 정당이 미지정/무소속이면, **When** 컬러를 적용하면, **Then** 시스템은 회색(#888) 폴백을 사용하고 경고 로그를 남긴다.

### User Story 3 — 다인 후보 비교를 2×2 그리드로 시각화 (Priority: P3)

크리에이터가 한 선거구의 후보 3~4명을 비교하는 영상을 만들고 싶을 때, 시스템은 **2×2 그리드 레이아웃**으로 후보 4명의 사진과 이름을 배치하고, 씬 시간에 따라 비교 데이터(재산/세금/공약 등)를 빨간 강조 폰트로 슬라이드 인한다.

**Why this priority**: 채널에서 가장 정교한 포맷이며, 데이터 시각화가 필요해 구현 난도가 가장 높다. 핵심 차별점이긴 하나 P1/P2 없이는 의미가 적다.

**Independent Test**: "평택을 후보 4명 비교" 주제 입력 → 2×2 그리드 + 빨강 데이터 슬라이드 영상 1편 출력 → 4개 후보 사진·이름·재산 금액 정확 표시 확인.

**Acceptance Scenarios**:

1. **Given** 3~4인 후보 비교 주제가 입력되면, **When** 시스템이 기획안을 생성하면, **Then** 본편 씬의 레이아웃이 `grid_2x2`로 자동 분류된다.
2. **Given** 그리드 레이아웃이 적용된 씬이 있을 때, **When** 비교 데이터(재산/세금 등)가 포함되면, **Then** 시스템은 데이터를 빨간색 큰 폰트로 페이드 인 애니메이션과 함께 표시한다.
3. **Given** 4명 중 일부의 인물 사진을 못 찾으면, **When** 그리드를 렌더링하면, **Then** 시스템은 해당 위치에 회색 실루엣 + 이름만 표시하고 영상 생성은 계속한다.

---

### User Story 4 — 1인 인물 데이터 카드 강조 (Priority: P3)

크리에이터가 한 인물의 핵심 데이터(예: "5년간 0원" 세금 미납, "127억" 재산 등)를 강조하고 싶을 때, 시스템은 **단일 인물 데이터 카드 레이아웃**(인물 사진 큰 컷 + 빨간 큰 숫자 강조)을 자동 적용한다.

**Why this priority**: 데이터 충격 패턴(채널의 강력한 후크). `grid_2x2`의 단일 인물 버전이라 구현은 가벼우나 P1-P3 다음 순위.

**Independent Test**: "조국 재산 56억 5년간 0원" 주제 입력 → 인물 사진 + 빨간 "56억" / "5년간 0원" 데이터 카드 영상 1편 출력.

**Acceptance Scenarios**:

1. **Given** 1인 + 수치 데이터 강조 주제가 입력되면, **When** 시스템이 분류하면, **Then** 씬 레이아웃이 `data_card`로 분류된다.
2. **Given** 데이터 강조 색상이 미지정이면, **When** 렌더링하면, **Then** 기본값 빨강(#E61E2B)이 적용된다.

---

### Edge Cases

- 입력 YouTube URL이 비공개·삭제·지역제한이면 → "영상을 가져올 수 없습니다" 메시지 후 종료
- 자막이 없고 음성 인식도 실패하면 → "자막 추출 실패, 주제 입력 모드로 다시 시도하세요" 안내
- 인물 카드 검색에서 동명이인이 섞이면 → 첫 검색 결과 사용 + 사용자가 영상 확인 후 재생성 가능
- 정당 컬러 매핑에 신생 정당이 없으면 → 회색(#888) 폴백 + 콘솔 경고
- LLM이 레이아웃을 오분류하면(인터뷰를 그리드로) → 사용자가 검수 화면에서 드롭다운으로 수동 변경 가능
- 영상 길이가 60초를 초과하면 → 자동으로 60초로 트리밍, 사용자에게 경고
- 메인 페이지에서 V3 버튼을 못 봐서 못 들어가면 → 헤더 영역에 항상 노출되는 버튼이라 발생 안 함 (위치 명시 필수)
- V1/V2 영상 생성 중 V3를 동시 실행하면 → 격리되어 있으므로 서로 영향 없음
- V3 생성 영상을 V1/V2 편집 화면에서 열면 → 데이터 모델이 다르므로 오류 메시지 + 거부

## Requirements *(mandatory)*

### Functional Requirements

#### 진입 및 격리 (Isolation Boundary)

- **FR-001**: 시스템은 메인 페이지(`/`)의 헤더 영역에 **"정치 V3" 진입 버튼 1개**를 노출해야 한다.
- **FR-002**: V3 기능은 별도 URL 경로(`/jpolitics`)에서 동작해야 한다. 메인 페이지의 기존 8개 탭(`image`/`manual`/`url`/`topic`/`political`/`political_pro`/`natv_clip`/`celebrity`)과 동시 사용 시 서로 간섭하지 않아야 한다.
- **FR-003**: V3 영상 생성은 기존 V1/V2 파이프라인 코드를 **수정 없이** 사용해야 한다(독립 모듈만 추가 가능). 시스템은 기존 297+ 회귀 테스트가 모두 통과해야 한다.
- **FR-004**: V3가 출력하는 영상 파일은 기존 출력 경로와 분리된 `data/jpolitics/` 디렉토리에 저장되어야 한다.

#### 입력 (Input)

- **FR-005**: 사용자는 V3에서 **YouTube URL 모드** 또는 **주제(topic) 입력 모드** 중 하나를 선택할 수 있어야 한다.
- **FR-006**: YouTube URL 모드는 정치 관련 YouTube 영상 1개의 URL을 받아 자막/음성 텍스트를 자동 추출해야 한다.
- **FR-007**: 주제 입력 모드는 주제 텍스트(필수), 톤(선택, 기본값 "분노·격앙"), 상세 설명(선택)을 받아야 한다.

#### 기획안 생성 (Plan Generation)

- **FR-008**: 시스템은 입력에 대해 정확히 **3개의 서로 다른 각도(angle)** 기획안을 생성해야 한다(예: title_anchor / audience_resonance / comparison).
- **FR-009**: 각 기획안은 (1) 주제·후크·핵심 클립 구간·이유, (2) 도입·중간·클라이맥스 흐름, (3) 씬별 나레이션, (4) CTA의 6요소를 포함해야 한다.
- **FR-010**: 시스템은 입력 자료(transcript 또는 주제)를 분석하여 영상의 핵심 구조를 4종 레이아웃 중 하나로 자동 분류해야 한다: `talking_head` / `vs_2way` / `comparison_grid` / `data_comparison`.
- **FR-011**: 시스템은 영상 전체에 표시될 **고정 헤드라인(headline_pin)** 을 8~14자 한글로 자동 생성해야 한다.
- **FR-012**: 사용자는 3개 기획안 중 1개를 선택할 수 있어야 한다.

#### 검수 (Review)

- **FR-013**: 사용자는 선택한 기획안의 씬별 자막, 레이아웃, 인물 카드를 영상 생성 전에 검수 화면에서 확인할 수 있어야 한다.
- **FR-014**: 사용자는 검수 화면에서 각 씬의 레이아웃을 4종 중 다른 것으로 수동 변경할 수 있어야 한다.
- **FR-015**: 사용자는 검수 화면에서 자막 텍스트, 강조 색(흰/노랑/빨강/파랑), 헤드라인 텍스트를 수정할 수 있어야 한다.

#### 영상 생성 (Video Rendering)

- **FR-016**: 시스템은 30~60초 길이의 9:16 세로 영상(MP4)을 생성해야 한다.
- **FR-017**: 모든 씬에 영상 상단 고정 노란 헤드라인이 표시되어야 한다(영상 전체 동안 같은 텍스트 유지).
- **FR-018**: 모든 씬에 하단 자막 박스가 표시되어야 한다(실시간 발언 캡션, 흰 라이트박스 + 검정 글자 또는 강조색).
- **FR-019**: 외부 영상 인용 시 하단에 "출처 : XXX / YYYY.MM.DD" 라벨이 표시되어야 한다.
- **FR-020**: 시스템은 채널 워터마크를 **표시하지 않아야 한다**(사용자 lock-in: 제외).
- **FR-021**: 시스템은 TTS 보이스로 **`ko-KR-InJoonNeural` +22%만** 사용해야 한다(V1 락인 유지, Gemini Charon 등 다른 보이스 불가).
- **FR-022**: 레이아웃이 `vs_card` 또는 `grid_2x2` 또는 `data_card`인 씬에는 인물 카드(사진 + 이름 + 소속 정당 컬러)가 표시되어야 한다.
- **FR-023**: 비교 데이터(예: "127억", "5년간 0원")는 기본 빨간색 큰 폰트로 강조 표시되어야 한다.

#### 인물 카드 및 정당 컬러

- **FR-024**: 시스템은 한국 정당명 → 헥스 컬러 매핑을 사전 정의된 표로 관리해야 한다(더불어민주당 #004EA2 / 국민의힘 #E61E2B / 조국혁신당 #0073CF / 개혁신당 #FF7920 / 무소속 #888888 등).
- **FR-025**: 시스템은 인물 이름으로 정면 단독 사진을 외부 이미지 검색에서 자동 페치해야 한다.
- **FR-026**: 인물 카드 검색 결과는 캐시(`data/politician_cards/{name}.json`)되어 재호출 시 즉시 반환되어야 한다.
- **FR-027**: 인물 사진 검색 실패 시 시스템은 회색 실루엣 + 이름 텍스트로 폴백해야 한다.
- **FR-028**: 정당이 매핑 표에 없으면 시스템은 회색(#888)으로 폴백하고 경고 로그를 남겨야 한다.

#### 콘텐츠 가드 (Compliance)

- **FR-029**: 시스템은 V3에서 생성된 영상의 자동 YouTube/TikTok 업로드 UI를 **표시하지 않아야 한다**(V2와 동일).
- **FR-030**: 시스템은 V3 출력 페이지 상단에 "**검수 필수: 사실 확인·법적 책임은 게시자에게 있음**" 경고 배너를 표시해야 한다.
- **FR-031**: 시스템은 인용한 YouTube 영상의 원본 URL을 영상 메타데이터에 보존해야 한다.

#### 출력 부가물

- **FR-032**: 시스템은 영상 1편 생성 시마다 **3줄 요약 + 해시태그 5~10개**를 함께 출력해야 한다(고정 룰, 사용자 메모리).
- **FR-033**: 시스템은 영상 생성 결과 페이지에 영상 다운로드 링크, 원본 기획안 JSON 링크, 3줄 요약·해시태그를 모두 노출해야 한다.

#### 효과음·전환·TTS·영상 추출 락인 (사용자 lock-in 2026-06-05)

- **FR-034**: 시스템은 영상에 **효과음(SFX)을 절대 삽입하지 않아야 한다**. TTS 음성과 (옵션) 클립 원본 오디오만 사용한다. 어떤 씬에도 sfx 트리거·BGM 페이드인 등 삽입 금지.
- **FR-035**: 시스템은 씬과 씬 사이에 **그라데이션 페이드/디졸브/플래시 전환을 적용하지 않아야 한다**. 모든 씬 전환은 **하드 컷(직접 컷)** 으로 처리한다. 첫 씬 등장·마지막 Outro 등장 시에도 fade-in/fade-out 미사용.
- **FR-036**: 시스템은 TTS 씬 음성 합성 시 **씬과 씬 사이 무음 구간(gap)을 0.3초로 고정**해야 한다 (300 ms). 자동 분할된 자막 그룹(`group_id`) 내부는 0초로 연속 합성, 그룹 경계에서만 0.3초 gap 적용.
- **FR-037**: 시스템은 영상 추출(클립 다운로드) 시 다음 흐름을 따라야 한다:
  1. 입력된 YouTube URL을 **Gemini Files API에 업로드**하여 영상/자막을 멀티모달 분석(transcript + 핵심 발언 timestamp 추출).
  2. **Claude가 Gemini 분석 결과를 받아** 씬별 영상 검색 키워드(`clip_search_query`)를 결정.
  3. **Claude가 결정한 키워드로 yt-dlp `ytsearch1`**로 추가 보조 클립 또는 원본 클립을 검색·다운로드(레이아웃이 `talking_head`이면 원본 클립의 timestamp 구간만 cut, `vs_card`/`grid_2x2`/`data_card`이면 인물 카드 + 보조 클립 검색).
  4. 클립 크롭은 9:16 letterbox (기존 V2와 동일 `youtube_news_searcher.cut_scene_clip` read-only import).

### Key Entities

- **JpoliticsScript**: V3 전용 쇼츠 스크립트 1편. 속성: `metadata`(제목·소스·소스 URL·소스 라벨), `scenes`(JpoliticsScene 리스트, 3~10개), `audio`(TTS 설정 락인), `background`(그라데이션 설정).
- **JpoliticsScene**: 단일 씬 단위. 속성: `id`, `timestamp`, `duration`(≤5초), `type`(title/body/comment), `text`(자막), `voice_text`(TTS 원문), `visual_layout`(`normal`/`vs_card`/`grid_2x2`/`data_card`), `subtitle_color`, `subtitle_emphasis`, `headline_pin`(첫 씬만 설정), `comparison_cards`(인물 카드 리스트, 옵셔널), `data_emphasis_color`, `clip_path`(원본 클립 경로), `clip_query`(검색어).
- **PoliticianCard**: 한 정치인의 시각 자료 1세트. 속성: `name`, `party`(소속 정당명), `party_color`(헥스), `photo_path`(로컬 사진 파일 경로), `data_label`(옵셔널, 예: "재산"), `data_value`(옵셔널, 예: "127억").
- **JpoliticsPlan**: 기획안 1개. 속성: `rank`(1~3), `angle`(title_anchor/audience_resonance/comparison), `format_type`(A/B/C), `layout_classification`(talking_head/vs_2way/comparison_grid/data_comparison), `topic`, `hook`, `clip_section`, `reason`, `flow_intro/middle/climax`, `narrations`(씬별 텍스트 리스트), `cta`, `youtube_search_keywords`(주제 모드만).
- **JpoliticsThreePlansResult**: 3개 기획안 묶음 + 입력 메타데이터(YouTube URL, 영상 제목, 영상 길이, transcript 경로).

## Assumptions

- 사용자는 한국 정치 콘텐츠 크리에이터이며, 결과 영상은 **검수 후 본인 책임**으로 게시한다.
- 인물 사진은 한국 포털의 이미지 검색을 기본 소스로 사용한다(이미 셀럽 모드용 API 키 보유).
- 정치인 이름 → 정당 매핑은 LLM 추론 + 사전 정의 표로 충분하며, 매핑 표는 운영 중 수동 갱신한다.
- 입력 YouTube 영상의 자막은 한국어이며, 자막이 없으면 음성 인식으로 폴백한다.
- 영상 1편 길이는 30~60초이며, 60초 초과 시 자동 트리밍한다.
- TTS 보이스 `ko-KR-InJoonNeural` +22%는 V1에서 검증된 락인 포맷으로 변경 불가(사용자 메모리).
- V3 출력 영상은 학습·논평·보도 목적이며, 자동 업로드는 영구히 차단한다(V2와 동일 정책).
- 메인 페이지 진입 버튼은 헤더 영역에 항상 노출되며, 기존 8개 탭과 시각적으로 구분되도록 amber 색상 배지를 사용한다.
- 효과음/전환 효과/BGM은 V3에서 영구 제외한다. 정치 콘텐츠는 텍스트·음성 전달력이 핵심이며 효과음·전환은 산만함을 유발한다고 판단.
- TTS 씬 간 0.3초 gap은 자연스러운 호흡 + 자막 인지 시간 확보의 균형점이다. 0초는 너무 빠르고 1초는 답답함.
- 영상 추출에서 Gemini는 멀티모달 분석 강점, Claude는 추론·검색 키워드 결정 강점을 각각 사용. 분업으로 정확도 향상 + Claude만 사용보다 transcript 추출 비용 감소.

## Out of Scope

- V1/V2 파이프라인의 수정 또는 V3 패턴 역적용
- V3 영상의 자동 YouTube/TikTok 업로드
- 1편 영상 안에서 4종 레이아웃을 임의로 혼합 사용(씬 단위 분류는 가능하나, "씬 1=grid, 씬 2=vs_card, 씬 3=data_card" 같은 자유 혼합은 v2 이후 검토)
- 정치인 이름 동명이인 자동 식별(사용자가 결과 확인 후 재생성)
- 영어/일본어 등 비한국어 정치 콘텐츠
- 실시간 라이브 방송 분석
- 페이스북·인스타그램 등 비-YouTube 소스
- 인물 카드의 사용자 업로드 사진(자동 페치만 지원)
- 영상 내 광고·BGM 자동 삽입(V1/V2 BGM 시스템 사용 안 함)

## Dependencies

- **공식 YouTube 영상 처리 도구**: 영상 다운로드 + 자막 추출(이미 프로젝트 보유)
- **외부 이미지 검색 API**: 인물 사진 자동 페치(이미 셀럽 모드용 API 키 보유)
- **LLM 분석 백엔드**: 기획안 생성(이미 정치_pro에서 사용 중)
- **음성 합성(TTS)**: 한국어 보이스 `ko-KR-InJoonNeural` 지원(이미 V1에서 사용 중)
- **영상 렌더링 엔진**: 9:16 세로 영상 출력(이미 V1/V2에서 사용 중인 엔진을 격리 모드로 별도 인스턴스 운영)
