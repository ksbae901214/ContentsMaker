# Feature Specification: Political Shorts Planner (정치 숏츠 기획자)

**Feature Branch**: `009-political-pro-planner`
**Created**: 2026-05-13
**Status**: Draft
**Input**: 정치 YouTube 영상을 입력하면 RTF 영상생성지침의 6요소 구조를 따르는 3개 기획안을 비교 제시하고, 사용자가 1개 선택 → 스크립트 검수 → 원본 9:16 클립 + Gemini TTS 음성으로 최종 쇼츠를 생성한다. 참고 채널: https://www.youtube.com/@김정치입니다 (강한 후킹·짧은 호흡·정치 풍자 톤, 단 정치 중립 유지).

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 3개 기획안 비교 후 1개 선택 (Priority: P1)

콘텐츠 제작자가 정치 YouTube 영상 URL 하나를 입력하면, 시스템은 그 영상에서 도출 가능한 **3가지 서로 다른 관점의 숏츠 기획안**을 동시에 제시한다. 각 기획안은 RTF 지침의 6요소(주제·후킹·사용 구간+이유·구성 흐름·타이밍별 나레이션·CTA)를 모두 포함하며, 제작자는 카드 형태로 비교한 뒤 가장 마음에 드는 1개를 선택한다.

**Why this priority**: 이 기능의 핵심 가치이자 기존 자동 클립 모드와의 결정적 차별점. 자동 선택 1개로는 "어떤 angle이 가장 잘 먹힐까"를 비교할 수 없어 매번 결과를 폐기하고 재생성하던 비효율을 해결한다. 이 단계 없이는 전체 기능이 무의미하다.

**Independent Test**: 정치 YouTube URL을 입력해 3개 기획안 카드가 표시되고, 각 카드가 6요소를 모두 포함하며, 3개의 angle이 서로 다르다는 점을 UI에서 직접 확인하면 검증 완료. 영상 생성까지 진행하지 않아도 이 단계만으로 "어떤 기획이 좋을지 비교 검토"라는 가치를 제공한다.

**Acceptance Scenarios**:

1. **Given** 자막이 있는 정치 YouTube URL을 입력했을 때, **When** "3 기획안 생성" 버튼을 누르면, **Then** 90초 이내에 3개 카드가 표시되고 각 카드에는 (1) 한 줄 주제 (2) 후킹 문구 (3) `mm:ss~mm:ss` 구간 + 선택 이유 (4) 시작-중간-클라이맥스 흐름 (5) 타이밍별 나레이션 목록 (6) CTA가 모두 보인다.
2. **Given** 3개 기획안이 표시된 상태에서, **When** 사용자가 한 카드의 "이 기획안으로 진행" 버튼을 누르면, **Then** 선택된 기획안이 씬 단위 스크립트로 변환되어 검수 화면으로 넘어간다.
3. **Given** 자막이 없는 영상이라도, **When** 시스템이 자체 음성 인식으로 transcript를 확보할 수 있으면, **Then** 동일하게 3개 기획안이 생성된다.
4. **Given** 3개 기획안이 표시되었을 때, **When** 카드들의 angle(관점) 라벨을 비교하면, **Then** 3개가 서로 다른 관점(예: 영상 제목 직결 / 시청자 반응 공감 / 비교·대조)이다.

### User Story 2 - 변환된 스크립트 검수 및 영상 생성 (Priority: P1)

제작자가 기획안을 선택하면 시스템은 자동으로 그것을 씬 단위 스크립트로 변환한다. 제작자는 검수 화면에서 씬별 텍스트·나레이션·길이를 미리 보고 필요 시 수정한 뒤 "영상 생성" 버튼을 누른다. 시스템은 원본 YouTube 영상에서 선택 구간을 잘라 9:16 세로 비율로 변환하고, 지정된 아나운서 톤 음성을 합성하여 30~60초의 MP4 쇼츠를 출력한다.

**Why this priority**: 기획만 보고 영상이 안 나오면 도구로서 가치가 없다. 검수 단계는 자동 생성된 스크립트의 사실 오류·과장·편향을 사람이 마지막으로 차단하는 안전망이다. 두 단계(기획안 선택 + 스크립트 검수)를 통과해야 비로소 영상이 생성되어 사용자 신뢰가 확보된다.

**Independent Test**: 임의의 기획안 1개를 직접 데이터로 제공해 검수 화면 → 영상 생성까지 진행하면, 원본 클립 + 음성 나레이션이 합쳐진 9:16 MP4가 30~60초 길이로 출력되는지 확인.

**Acceptance Scenarios**:

1. **Given** 기획안 1개를 선택했을 때, **When** 시스템이 스크립트 변환을 마치면, **Then** 검수 화면에 씬 목록·각 씬의 텍스트·나레이션·예상 길이가 표시된다.
2. **Given** 검수 화면에서 씬 텍스트를 수정했을 때, **When** "영상 생성" 버튼을 누르면, **Then** 수정된 내용이 반영된 최종 영상이 만들어진다.
3. **Given** 검수를 완료하고 영상 생성을 시작했을 때, **When** 처리가 끝나면, **Then** 9:16 세로 비율의 MP4 파일이 30~60초 길이로 출력되고 각 씬은 5초를 넘지 않는다.
4. **Given** 영상이 생성되었을 때, **When** 결과 파일을 재생하면, **Then** 원본 YouTube 영상의 발언 구간(9:16 변환)과 합성된 아나운서 톤 음성 나레이션이 함께 들린다.

### User Story 3 - 정치 중립성 보장 (Priority: P2)

기획안 생성 시스템은 "특정 정당·정치인을 지지하거나 비판하는 표현"을 만들지 않도록 사전에 가이드된다. 자극적 후킹은 허용하되 왜곡·의견·추측·루머는 금지된다. 출력에는 영상에서 직접 확인 가능한 사실만 포함된다.

**Why this priority**: 정치 콘텐츠 영역에서 편향은 곧 플랫폼 신고·법적 리스크·채널 정지로 직결된다. P1은 아니지만(중립성 가드 없이도 기능 자체는 동작) 운영 위험을 줄이기 위해 출시 전 반드시 갖춰야 하는 가드레일이다.

**Independent Test**: 시스템이 자동 생성기에 주는 지시문을 검사하여 4가지 절대 준수 항목(사실만 / 의견금지 / 편향금지 / 왜곡금지)이 명시적으로 포함되어 있는지 자동 테스트로 확인. 또한 샘플 영상 3개로 생성한 기획안 9개를 사람이 검토했을 때 명시적 편향 표현이 없어야 한다.

**Acceptance Scenarios**:

1. **Given** 시스템이 자동 생성기에 보내는 기획안 지시문을 검사할 때, **When** 지시문 내용을 확인하면, **Then** "영상에서 확인 가능한 사실만 사용", "개인 의견·해석·추측·루머 금지", "특정 정당·정치인 지지/비판 금지", "왜곡 금지" 4문구가 모두 포함되어 있다.
2. **Given** 생성된 3개 기획안의 후킹·나레이션·CTA를 사람이 읽었을 때, **When** 텍스트를 검토하면, **Then** 특정 정당명·정치인을 직접 지지하거나 비판하는 평가 표현이 없다(사실 인용·중립적 묘사만 포함).
3. **Given** 최종 영상 화면 또는 결과 안내에서, **When** 사용자에게 보이는 메시지를 확인하면, **Then** "출력은 자동 생성 결과이며 게시 전 사용자 검수가 필요합니다" 취지의 경고가 표시된다.

---

### Edge Cases

- **자막·음성 인식 모두 실패**: 시스템은 명시적 오류 메시지("자막·음성 인식 모두 실패")를 표시하고 다른 URL을 시도하도록 안내한다. 환각된 기획안을 생성하지 않는다.
- **transcript는 있으나 의미 있는 발언 구간이 없음(예: 음악만 흐르는 영상)**: "의미 있는 발언 구간을 찾지 못했습니다" 오류를 반환하고 빈 내용으로 기획안을 만들지 않는다.
- **YouTube 영상이 비공개·삭제·지역 제한**: 다운로드 단계에서 실패하면 즉시 알리고 다른 URL을 안내한다.
- **영상이 너무 짧음(30초 미만)**: 사용 구간이 영상 길이를 넘어가지 않도록 자동 클램프되며, 그래도 30초 이상의 쇼츠를 못 만들면 경고 메시지를 표시한다.
- **영상이 너무 김(2시간 이상)**: 다운로드·transcript 추출이 90초 한도를 초과할 수 있다. 시스템은 다운로드 시작 시점에 영상 길이를 확인하고 사용자에게 처리 시간 추정치를 알린다.
- **기획안 3개 형식이 어긋남**: 자동으로 1회 재시도하고, 그래도 실패하면 사용자에게 명시적 오류를 알린다.
- **사용 구간 합계가 60초 초과**: 스크립트 변환 단계에서 5초 이하 씬으로 자동 분할되어 영상 총 길이 30~60초를 유지한다.
- **음성 합성 실패(자격 증명 없음·할당량 초과)**: 사용자에게 명시적 안내와 함께 영상 생성을 중단한다(빈 음성으로 진행하지 않는다).

## Requirements *(mandatory)*

### Functional Requirements

#### 입력 및 영상 준비
- **FR-001**: 시스템은 사용자가 "정치 기획" 탭에서 YouTube URL 1개를 입력할 수 있도록 한다.
- **FR-002**: 시스템은 입력된 YouTube 영상의 음성 텍스트(transcript)를 확보한다. 영상 자체 자막이 있으면 우선 사용하고, 없으면 자동 음성 인식으로 대체한다.
- **FR-003**: 자막·음성 인식 모두 실패한 경우 시스템은 환각된 기획안을 만들지 않고 명시적 오류를 반환한다.

#### 3 기획안 생성
- **FR-004**: 시스템은 transcript와 영상 제목을 바탕으로 한 번에 3개의 숏츠 기획안을 생성한다.
- **FR-005**: 각 기획안은 다음 6요소를 모두 포함해야 한다 — (a) 한 줄 주제, (b) 후킹 문구, (c) 사용 구간(`mm:ss~mm:ss` + 선택 이유), (d) 시작·중간·클라이맥스 흐름, (e) 타이밍별 나레이션 목록, (f) 마무리 유도 문구(CTA).
- **FR-006**: 3개 기획안은 서로 다른 관점(angle)을 가져야 한다(예: 영상 제목과 직결 / 시청자 반응 공감 / 비교·대조).
- **FR-007**: 기획안 생성 지시문에는 다음 4가지 절대 준수 항목이 명시적으로 포함되어야 한다 — "영상에서 확인 가능한 사실만 사용", "개인 의견·해석·추측·루머 금지", "정치적 편향(특정 정당·정치인 지지/비판) 금지", "자극은 허용·왜곡은 금지".
- **FR-008**: 시스템은 3개 기획안 생성에 실패하면 자동으로 1회 재시도하고, 그래도 실패하면 사용자에게 명시적 오류를 알린다.

#### 사용자 선택 및 스크립트 변환
- **FR-009**: 시스템은 3개 기획안을 카드 형태로 동시에 비교 표시한다.
- **FR-010**: 사용자는 3개 중 정확히 1개를 선택하여 다음 단계로 진행할 수 있다.
- **FR-011**: 사용자가 1개를 선택하면 시스템은 그 기획안을 씬 단위 스크립트로 자동 변환한다.
- **FR-012**: 변환된 스크립트의 각 씬은 5.0초를 넘지 않도록 자동 분할된다.
- **FR-013**: 사용자가 선택한 사용 구간이 영상 길이를 초과하지 않도록 자동 클램프된다.

#### 스크립트 검수
- **FR-014**: 시스템은 변환된 스크립트를 검수 화면에 표시하여 사용자가 씬별 텍스트·나레이션·길이를 검토하고 수정할 수 있도록 한다.
- **FR-015**: 사용자가 검수를 완료하고 "영상 생성" 버튼을 누르기 전까지는 영상 생성 작업이 시작되지 않는다.

#### 영상 생성
- **FR-016**: 시스템은 원본 YouTube 영상에서 선택된 구간을 잘라 9:16 세로 비율로 변환하여 영상 소스로 사용한다(AI 영상 생성기는 사용하지 않는다).
- **FR-017**: 시스템은 검수 완료된 스크립트의 나레이션을 지정된 아나운서 톤(빠른 페이스) 음성 설정으로 합성한다.
- **FR-018**: 최종 영상은 9:16 세로 비율의 MP4이며 길이는 30~60초이다.
- **FR-019**: 음성 합성이 실패하면 시스템은 빈 음성으로 진행하지 않고 사용자에게 명시적 오류를 알린다.
- **FR-020**: 최종 영상 출력은 사용자가 별도 트리거를 누르기 전까지 외부 플랫폼(YouTube/TikTok 등)에 자동 업로드되지 않는다.

#### 안전성 및 안내
- **FR-021**: 최종 결과 화면에는 "출력은 자동 생성 결과이며 게시 전 사용자 검수가 필요합니다"는 취지의 경고가 표시되어야 한다.
- **FR-022**: 시스템은 변동 비용을 발생시키지 않는다(원본 YouTube 클립 사용 + 무료 한도 내 음성 합성).

### Key Entities *(include if feature involves data)*

- **ShortsPlan(숏츠 기획안)**: 영상생성지침의 6요소(주제·후킹·구간·이유·흐름·나레이션·CTA)와 angle(관점) 메타데이터를 담는 불변 객체. 한 번의 생성에서 정확히 3개가 만들어지며 각각은 서로 다른 angle을 가진다.
- **Narration(타이밍별 나레이션 항목)**: ShortsPlan 안에 여러 개 포함되는 항목으로 (시작 시각, 종료 시각, 대사) 3요소로 구성된다.
- **ShortsScript(쇼츠 스크립트)**: 기존 시스템 공통 데이터 모델. ShortsPlan을 사용자가 선택하면 이 형식으로 변환되어 검수·렌더 단계에서 사용된다. source_type을 "political_pro"로 설정한다.
- **VideoClip(원본 클립)**: 원본 YouTube 영상에서 잘려나온 9:16 변환 영상 파일.
- **Transcript(음성 텍스트)**: 영상에서 추출한 (시작, 종료, 텍스트) 항목의 시계열 목록.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 정치 YouTube URL을 입력한 후 3개 기획안 카드를 보기까지 평균 90초 이내(다운로드·음성 인식·기획 생성 합계).
- **SC-002**: 생성된 3개 기획안 중 어떤 카드를 골라도, 사용자는 추가 입력 없이 5분 이내(검수 시간 포함, 영상 처리 시간 제외)에 최종 쇼츠 영상까지 도달할 수 있다.
- **SC-003**: 최종 영상의 길이는 30~60초 범위 안에 있고, 모든 씬은 5초를 넘지 않는다(100% 충족).
- **SC-004**: 자체 테스트 영상 5개로 측정 시, 5건 모두에서 3개 기획안이 정상 생성된다(자막 또는 음성 인식이 가능한 영상 기준 100% 성공률).
- **SC-005**: 자동 검사로 기획안 생성 지시문에서 4가지 절대 준수 항목이 모두 발견된다(100%).
- **SC-006**: 샘플 영상 3개로 생성된 9개 기획안 중 사람 검토에서 명시적 정당·정치인 지지/비판 표현이 발견되는 비율은 0%이다.
- **SC-007**: 최종 영상의 변동 단가는 0원이다(원본 클립 사용 + 무료 한도 내 음성).
- **SC-008**: 음성 합성·다운로드 실패 등 오류 발생 시, 100%의 경우에서 빈 콘텐츠가 아닌 명시적 오류 메시지가 사용자에게 표시된다.

## Assumptions

- 입력 영상은 한국어 발화를 포함하는 정치 콘텐츠라고 가정한다(다국어 영상은 이번 단계 범위 밖).
- 사용자는 입력 영상이 자신의 사용 권한 범위에 있다고 확인했다고 가정한다(저작권·초상권 검토는 사용자 책임).
- "시청자 반응 공감" 관점 기획안은 실제 댓글을 수집하지 않고 transcript와 영상 제목을 기반으로 일반화된 시청자 반응을 추정한다.
- 사용 구간의 시작·종료 시각은 영상 내 실제 발언 위치를 정확히 반영해야 하지만, 자막 또는 음성 인식 결과의 타임스탬프 정확도(±1~2초)에 의존한다.
- 검수 화면은 기존 시스템에 존재하는 스크립트 검수 화면을 그대로 활용한다.
- 음성 합성은 무료 한도 안에서 운영되며, 한도 초과 상황은 운영자가 별도 모니터링한다.

## Out of Scope

- 후처리 LLM(별도 모델)에 의한 중립성 검증 단계.
- 영상의 실제 댓글 수집·분석.
- 과거 발언·뉴스 클립 등 비교 영상의 자동 첨부.
- YouTube/TikTok 등 외부 플랫폼으로의 자동 업로드(생성 완료까지만, 업로드 트리거는 사용자 수동).
- AI 영상 생성(텍스트→영상). 본 기능은 원본 YouTube 클립만 사용한다.
- 다국어 영상 지원.
