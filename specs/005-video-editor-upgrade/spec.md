# Feature Specification: 영상 제작/편집 기능 고도화

**Feature Branch**: `005-video-editor-upgrade`
**Created**: 2026-03-26
**Status**: Draft
**Input**: 편집 고도화(Vrew 참고) + AI 영상 클립 통합(Seedance 2.0) + 고급 편집 + 확장 기능

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 타임라인 기반 씬 편집 (Priority: P1)

사용자가 생성된 영상의 씬을 타임라인 UI에서 직관적으로 편집한다. 씬 순서 변경, 길이 조절, 분할, 병합이 가능하며, 편집 결과를 즉시 확인할 수 있다.

**Why this priority**: 현재 카드 목록 방식은 씬 간 관계와 전체 흐름을 파악하기 어렵다. 타임라인 편집은 영상 편집의 핵심 UX이며, 이후 모든 기능(트랜지션, 효과음, AI 영상 클립)의 기반이 된다.

**Independent Test**: 기존 생성된 영상의 씬을 타임라인에서 순서 변경, 길이 조절, 분할/병합한 후 리렌더링하여 결과가 정확히 반영되는지 확인.

**Acceptance Scenarios**:

1. **Given** 6개 씬으로 구성된 영상, **When** 타임라인에서 씬 3을 드래그하여 씬 1 뒤로 이동, **Then** 씬 순서가 1-3-2-4-5-6으로 변경되고 리렌더링 시 해당 순서로 재생됨
2. **Given** 8초짜리 씬, **When** 타임라인에서 씬 끝을 드래그하여 5초로 줄임, **Then** 씬 duration이 5초로 변경되고 후속 씬 타이밍이 자동 조정됨
3. **Given** "나는 선넘네라는 표현을 친한 친구한테도 안 쓰거든"이라는 텍스트의 씬, **When** "친한 친구" 앞에서 씬 분할, **Then** 두 개의 씬이 생성되고 각각 독립적인 voice_text를 가짐
4. **Given** 인접한 두 개의 씬, **When** 병합 버튼 클릭, **Then** 텍스트가 합쳐지고 duration이 합산된 하나의 씬이 됨
5. **Given** 편집 화면, **When** "카드 뷰"와 "타임라인 뷰" 토글, **Then** 동일한 씬 데이터가 두 가지 UI로 표시됨

---

### User Story 2 - 자막 스타일 커스터마이즈 (Priority: P1)

사용자가 각 씬의 자막 폰트, 크기, 색상, 위치, 배경 투명도를 자유롭게 설정한다. 프리셋 템플릿을 선택하거나 직접 커스터마이즈할 수 있다.

**Why this priority**: 자막은 Shorts 영상에서 가장 큰 비중을 차지하는 시각 요소다. 동일한 콘텐츠라도 자막 스타일에 따라 조회수와 시청 유지율이 크게 달라진다.

**Independent Test**: 영상의 자막 스타일을 변경(폰트, 색상, 위치)한 후 렌더링하여 변경사항이 영상에 정확히 반영되는지 확인.

**Acceptance Scenarios**:

1. **Given** 영상 편집 화면, **When** 자막 프리셋 "뉴스형" 선택, **Then** 모든 씬의 자막이 뉴스 앵커 스타일(하단 배치, 검정 배경 바, 흰색 텍스트)로 변경됨
2. **Given** 특정 씬 선택, **When** 자막 색상을 노란색, 크기를 72px로 변경, **Then** 해당 씬만 변경된 스타일로 표시되고 다른 씬은 유지됨
3. **Given** 자막 위치 설정, **When** 드래그로 화면 상단 30% 위치로 이동, **Then** 렌더링된 영상에서 자막이 해당 위치에 표시됨
4. **Given** 커스텀 자막 스타일, **When** "내 스타일로 저장" 클릭, **Then** 프리셋 목록에 사용자 정의 스타일이 추가됨

---

### User Story 3 - AI 영상 클립 생성 (Priority: P2)

사용자가 정적 이미지 대신 AI 생성 영상 클립을 씬 배경으로 사용한다. 이미지 모드, 영상 모드, 하이브리드 모드 중 선택할 수 있다.

**Why this priority**: 동적 영상 클립은 정적 이미지 대비 시청 유지율을 크게 높이지만, 추가 비용이 발생한다. 편집 기반(P1)이 먼저 갖춰져야 영상 클립을 효과적으로 활용할 수 있다.

**Independent Test**: "영상 모드"를 선택하여 영상을 생성하고, 완성된 MP4에서 각 씬에 움직이는 영상 클립이 배경으로 재생되는지 확인.

**Acceptance Scenarios**:

1. **Given** 영상 생성 화면, **When** "비주얼 모드"에서 "AI 영상 클립" 선택 후 생성, **Then** 각 씬에 AI가 생성한 짧은 영상 클립이 배경으로 재생됨
2. **Given** 영상 생성 화면, **When** "하이브리드" 모드 선택, **Then** 제목/댓글 씬은 정적 이미지, 본문 씬은 영상 클립으로 생성됨
3. **Given** 기존에 GPT Image로 생성한 정적 이미지, **When** 씬 편집에서 "영상으로 변환" 클릭, **Then** 해당 이미지를 기반으로 움직이는 영상 클립이 생성됨
4. **Given** AI 영상 클립 생성 중, **When** 생성이 실패하거나 타임아웃, **Then** 자동으로 정적 이미지로 폴백되고 사용자에게 알림 표시
5. **Given** 영상 모드로 6개 씬 생성, **When** 생성 진행 중, **Then** 각 씬별 생성 상태(대기중/생성중/완료/실패)가 실시간으로 표시됨

### User Story 4 - 트랜지션 효과 (Priority: P2)

사용자가 씬 전환 시 다양한 트랜지션 효과(slide, zoom, dissolve, wipe 등)를 적용한다.

**Why this priority**: 씬 전환 효과는 영상의 완성도를 크게 높이는 요소이며, 구현 범위가 명확하고 독립적이다.

**Independent Test**: 각 씬 사이에 서로 다른 트랜지션을 적용한 후 렌더링하여 전환 효과가 영상에서 재생되는지 확인.

**Acceptance Scenarios**:

1. **Given** 인접한 두 씬, **When** 트랜지션 "slide left" 선택, **Then** 씬 전환 시 왼쪽으로 밀리는 애니메이션이 재생됨
2. **Given** 트랜지션 설정 화면, **When** 효과 종류와 지속 시간(0.3~1.0초) 선택, **Then** 선택한 효과와 시간으로 전환이 재생됨
3. **Given** 6개 씬 영상, **When** "전체 적용" 버튼으로 모든 씬에 동일 트랜지션 적용, **Then** 모든 씬 전환에 선택한 효과가 일괄 적용됨

---

### User Story 5 - 음성 선택 및 미리듣기 (Priority: P2)

사용자가 여러 AI 음성 중 원하는 음성을 선택하고 미리 들어본 후 적용한다.

**Why this priority**: 현재 단일 음성만 지원하여 콘텐츠 다양성이 제한된다. 음성 선택은 비용 추가 없이(edge-tts 무료) 영상 다양성을 높일 수 있다.

**Independent Test**: 음성 목록에서 다른 음성을 선택하고 미리듣기 후 적용하여, 렌더링된 영상에서 선택한 음성이 재생되는지 확인.

**Acceptance Scenarios**:

1. **Given** 음성 선택 화면, **When** 한국어 음성 목록 표시, **Then** 최소 5종 이상의 음성이 성별/톤 정보와 함께 표시됨
2. **Given** 음성 목록, **When** 특정 음성의 "미리듣기" 버튼 클릭, **Then** 짧은 샘플 문장이 해당 음성으로 즉시 재생됨
3. **Given** 새로운 음성 선택, **When** "적용" 클릭 후 리렌더링, **Then** 전체 영상의 TTS가 선택한 음성으로 변경됨

---

### User Story 6 - 실시간 미리보기 (Priority: P3)

사용자가 편집 내용을 리렌더링 없이 브라우저에서 즉시 미리보기한다. 텍스트, 자막 스타일, 트랜지션 변경이 실시간으로 반영된다.

**Why this priority**: 현재 모든 변경사항 확인에 전체 리렌더링(수분)이 필요하다. 실시간 미리보기는 편집 워크플로우를 획기적으로 개선하지만, 기반 편집 기능(P1, P2)이 먼저 필요하다.

**Independent Test**: 씬 텍스트를 수정하고 미리보기 플레이어에서 변경사항이 즉시 반영되는지 확인. 최종 MP4 export는 별도 "렌더링" 버튼으로 수행.

**Acceptance Scenarios**:

1. **Given** 편집 화면의 미리보기 플레이어, **When** 씬 텍스트 수정, **Then** 3초 이내에 플레이어에 변경된 텍스트가 반영됨
2. **Given** 미리보기 플레이어, **When** 재생 버튼 클릭, **Then** 편집 중인 영상이 브라우저에서 실시간 재생됨
3. **Given** 미리보기로 최종 확인 완료, **When** "최종 렌더링" 버튼 클릭, **Then** MP4 파일이 생성되어 다운로드 가능

---

### User Story 7 - 프로젝트 저장 및 불러오기 (Priority: P3)

사용자가 편집 중인 영상 프로젝트를 저장하고, 나중에 불러와서 이어서 편집한다.

**Why this priority**: 현재 생성 후 일회성 편집만 가능하여, 복잡한 편집 작업이 어렵다. 프로젝트 관리는 반복 편집의 핵심이다.

**Independent Test**: 영상을 편집하다가 프로젝트를 저장하고, 브라우저를 닫았다가 다시 열어 프로젝트를 불러오면 편집 상태가 그대로 복원되는지 확인.

**Acceptance Scenarios**:

1. **Given** 편집 중인 영상, **When** "프로젝트 저장" 클릭, **Then** 현재 상태(스크립트, 이미지, 스타일, 트랜지션)가 저장됨
2. **Given** 메인 화면, **When** "최근 프로젝트" 목록에서 프로젝트 선택, **Then** 저장 시점의 편집 상태가 그대로 복원됨
3. **Given** 저장된 프로젝트, **When** "삭제" 클릭 후 확인, **Then** 프로젝트와 관련 파일이 삭제됨

---

### User Story 8 - 효과음 삽입 (Priority: P3)

사용자가 씬에 효과음(놀람, 웃음, 박수 등)을 추가하여 영상의 몰입감을 높인다.

**Why this priority**: 효과음은 Shorts 영상에서 시청자 반응을 유도하는 중요 요소이지만, 핵심 편집 기능 이후에 추가해도 충분하다.

**Independent Test**: 특정 씬에 효과음을 추가한 후 렌더링하여 해당 타이밍에 효과음이 재생되는지 확인.

**Acceptance Scenarios**:

1. **Given** 씬 편집 화면, **When** 효과음 라이브러리에서 "놀람" 효과음 선택, **Then** 해당 씬 시작 시 효과음이 재생됨
2. **Given** 효과음이 추가된 씬, **When** 효과음 볼륨을 조절, **Then** TTS 음성 대비 적절한 볼륨으로 재생됨
3. **Given** 효과음 라이브러리, **When** 카테고리(놀람/웃음/감동/강조) 필터링, **Then** 해당 카테고리의 효과음만 표시됨

---

### User Story 9 - 일괄 생성 (Priority: P4)

사용자가 여러 게시글 URL이나 텍스트를 한 번에 입력하면 순차적으로 영상이 자동 생성된다.

**Why this priority**: 대량 콘텐츠 생산의 핵심이지만, 단일 영상의 품질과 편집 기능이 먼저 완성되어야 의미가 있다.

**Independent Test**: 3개의 URL을 큐에 추가하고 일괄 생성을 실행하여 3개의 영상이 순차적으로 생성되는지 확인.

**Acceptance Scenarios**:

1. **Given** 일괄 생성 화면, **When** 5개 URL을 입력하고 "일괄 생성" 클릭, **Then** 큐에 5개 작업이 추가되고 순차 처리가 시작됨
2. **Given** 일괄 생성 진행 중, **When** 큐 화면 확인, **Then** 각 작업의 상태(대기중/처리중/완료/실패)와 진행률이 표시됨
3. **Given** 일괄 생성 중 특정 URL 처리 실패, **When** 실패 발생, **Then** 해당 작업만 "실패"로 표시되고 나머지 작업은 계속 처리됨

---

### User Story 10 - 템플릿 시스템 (Priority: P4)

사용자가 자막 스타일, 트랜지션, 음성, BGM 조합을 "템플릿"으로 저장하고 재사용한다.

**Why this priority**: 반복적인 영상 생성에서 일관된 스타일 유지를 위한 편의 기능이다.

**Independent Test**: 템플릿을 생성하고 새 영상에 적용하여 저장된 스타일이 자동 적용되는지 확인.

**Acceptance Scenarios**:

1. **Given** 커스텀 자막/트랜지션/음성 설정, **When** "템플릿으로 저장" 클릭 후 이름 입력, **Then** 템플릿 목록에 추가됨
2. **Given** 새 영상 생성 시, **When** 템플릿 선택, **Then** 해당 템플릿의 모든 스타일 설정이 자동 적용됨
3. **Given** 기본 제공 템플릿 3개(유머형/감동형/뉴스형), **When** 첫 사용, **Then** 별도 설정 없이 바로 선택 가능

---

### User Story 11 - 다국어 자막 (Priority: P4)

사용자가 한국어 자막에 영어/일본어 번역 자막을 추가하여 해외 시청자도 타겟할 수 있다.

**Why this priority**: 해외 Shorts 채널 운영 지원. 핵심 기능 완성 후 확장 단계에서 구현.

**Independent Test**: 한국어 영상에 영어 자막 추가 옵션을 켜고 렌더링하여 이중 자막이 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 한국어 영상, **When** "영어 자막 추가" 옵션 ON 후 생성, **Then** 한국어 자막 아래에 영어 번역 자막이 함께 표시됨
2. **Given** 번역된 자막, **When** 영어 자막 텍스트를 수동 수정, **Then** 수정된 번역이 렌더링에 반영됨
3. **Given** 다국어 자막 설정, **When** 대상 언어로 영어/일본어 중 선택, **Then** 선택한 언어로 번역된 자막이 생성됨

---

### Edge Cases

- AI 영상 클립 생성에 5분 이상 소요되는 경우 → 타임아웃 후 정적 이미지로 자동 폴백, 사용자에게 알림
- 씬 분할 시 텍스트가 1단어만 남는 경우 → 최소 2단어 이상 필요하다는 경고 표시
- 씬 병합으로 텍스트가 45자를 초과하는 경우 → 초과 경고 표시, 사용자 확인 후 병합
- 트랜지션 duration이 씬 duration보다 긴 경우 → 자동으로 씬 길이의 50%로 제한
- 효과음과 TTS가 동시에 재생되어 음성이 안 들리는 경우 → 효과음 볼륨을 TTS 대비 20% 이하로 자동 제한
- 프로젝트 저장 후 원본 이미지 파일이 삭제된 경우 → "원본 파일 누락" 경고, 재생성 옵션 제공
- 일괄 생성 중 시스템 종료 → 완료된 작업은 보존, 미완료 작업은 "중단됨" 상태로 표시
- 번역 API 호출 실패 → 한국어 자막만으로 영상 생성, 번역 실패 알림
- AI 영상 생성 비용이 예산을 초과할 경우 → 생성 전 예상 비용 표시, 사용자 확인 후 진행

## Requirements *(mandatory)*

### Functional Requirements

#### Phase 1: 편집 고도화

- **FR-001**: 시스템은 타임라인 UI에서 씬을 수평 바로 표시하고, 드래그로 순서 변경과 길이 조절을 지원해야 한다
- **FR-002**: 사용자는 카드 뷰와 타임라인 뷰를 토글할 수 있어야 한다
- **FR-003**: 사용자는 씬을 텍스트 커서 위치 기준으로 두 개로 분할할 수 있어야 한다
- **FR-004**: 사용자는 인접한 두 씬을 하나로 병합할 수 있어야 한다
- **FR-005**: 사용자는 씬별로 자막 폰트, 크기, 색상, 위치, 배경 투명도를 설정할 수 있어야 한다
- **FR-006**: 시스템은 최소 3개 이상의 자막 프리셋 템플릿(뉴스형, 유머형, 감성형)을 제공해야 한다
- **FR-007**: 사용자는 커스텀 자막 스타일을 프리셋으로 저장할 수 있어야 한다
- **FR-008**: 시스템은 최소 5종의 씬 전환 효과(fade, slide, zoom, dissolve, wipe)를 제공해야 한다
- **FR-009**: 사용자는 트랜지션 효과의 지속 시간을 0.3~1.0초 범위에서 설정할 수 있어야 한다
- **FR-010**: 사용자는 한국어 TTS 음성 목록(최소 5종)에서 음성을 선택할 수 있어야 한다
- **FR-011**: 시스템은 선택한 음성의 미리듣기 기능을 제공해야 한다

#### Phase 2: AI 영상 클립 통합

- **FR-012**: 사용자는 영상 생성 시 비주얼 모드를 선택할 수 있어야 한다: 정적 이미지 / AI 영상 클립 / 하이브리드
- **FR-013**: 시스템은 Seedance 2.0을 기본 AI 영상 생성 도구로 사용해야 한다
- **FR-014**: 시스템은 기존 정적 이미지(GPT Image)를 입력으로 받아 영상 클립으로 변환(Image-to-Video)할 수 있어야 한다
- **FR-015**: AI 영상 생성 실패 시 자동으로 정적 이미지로 폴백해야 한다
- **FR-016**: 여러 씬의 영상 클립은 병렬로 생성하여 전체 생성 시간을 최소화해야 한다
- **FR-017**: 영상 생성 전 예상 비용을 사용자에게 표시하고 확인을 받아야 한다
- **FR-018**: 각 씬별 영상 생성 상태(대기중/생성중/완료/실패)를 실시간으로 표시해야 한다

#### Phase 3: 고급 편집

- **FR-019**: 사용자는 브라우저 내에서 편집 결과를 실시간으로 미리보기할 수 있어야 한다
- **FR-020**: 미리보기에서 변경사항은 3초 이내에 반영되어야 한다
- **FR-021**: 시스템은 카테고리별(놀람/웃음/감동/강조) 효과음 라이브러리를 제공해야 한다
- **FR-022**: 사용자는 씬에 효과음을 추가하고 볼륨을 조절할 수 있어야 한다
- **FR-023**: 사용자는 편집 상태를 프로젝트로 저장하고 나중에 불러올 수 있어야 한다
- **FR-024**: 메인 화면에 최근 프로젝트 목록이 표시되어야 한다

#### Phase 4: 확장 기능

- **FR-025**: 사용자는 여러 URL/텍스트를 큐에 추가하여 일괄 생성할 수 있어야 한다
- **FR-026**: 일괄 생성 시 개별 작업의 실패가 다른 작업에 영향을 주지 않아야 한다
- **FR-027**: 사용자는 자막 스타일 + 트랜지션 + 음성 + BGM 조합을 템플릿으로 저장/적용할 수 있어야 한다
- **FR-028**: 시스템은 한국어 자막을 영어/일본어로 번역하여 이중 자막을 지원해야 한다
- **FR-029**: 사용자는 번역된 자막을 수동으로 수정할 수 있어야 한다

### Key Entities

- **프로젝트**: 영상 편집의 전체 상태를 포함하는 단위. 스크립트, 이미지/영상 경로, 스타일 설정, 트랜지션 설정을 포함한다.
- **자막 스타일**: 폰트, 크기, 색상, 위치, 배경 투명도 등 자막의 시각적 표현. 씬별 또는 전체 적용 가능.
- **트랜지션**: 두 씬 사이의 전환 효과. 효과 종류와 지속 시간을 속성으로 가진다.
- **영상 클립**: AI가 생성한 짧은 영상. 프롬프트, 해상도, 길이, 비용, 원본 이미지(있을 경우)를 속성으로 가진다.
- **효과음**: 씬에 삽입되는 짧은 오디오. 카테고리, 재생 타이밍, 볼륨을 속성으로 가진다.
- **템플릿**: 자막 스타일 + 트랜지션 + 음성 + BGM의 조합. 이름과 프리뷰 정보를 가진다.
- **배치 작업**: 일괄 생성 큐의 개별 항목. 입력 데이터, 처리 상태, 결과물 경로를 가진다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 씬 순서 변경, 분할, 병합을 10초 이내에 완료할 수 있다 (현재: 불가능)
- **SC-002**: 자막 스타일 변경 후 결과 확인까지 걸리는 시간이 3초 이내이다 (실시간 미리보기 기준)
- **SC-003**: AI 영상 클립 모드로 생성한 영상의 시각적 움직임이 정적 이미지 대비 확인 가능하다
- **SC-004**: 트랜지션 효과 5종 이상이 렌더링된 영상에서 육안으로 구분 가능하다
- **SC-005**: 음성 변경이 3단계 이내(목록 열기 → 선택 → 적용)로 완료된다
- **SC-006**: 프로젝트 저장 후 불러오기 시 모든 편집 상태가 100% 복원된다
- **SC-007**: 일괄 생성에서 단일 실패가 전체 큐를 중단시키지 않는다 (격리율 100%)
- **SC-008**: AI 영상 생성 실패 시 정적 이미지 폴백이 자동으로 동작한다 (폴백 성공률 100%)
- **SC-009**: 영상 1개당 AI 영상 클립 추가 비용이 $2.00 이하이다 (5씬 기준, Seedance 사용)

## Assumptions

- Seedance 2.0 API는 안정적으로 제공되며 9:16 종횡비와 Image-to-Video 기능을 지원한다
- 효과음은 로열티프리 소스에서 수동으로 다운로드하여 프로젝트에 번들한다 (10~20개)
- 번역 기능은 기존 OpenAI GPT API를 활용한다 (별도 번역 API 불필요)
- 실시간 미리보기는 최종 렌더링과 100% 동일하지 않을 수 있다 (근사치 미리보기)
- 프로젝트 저장은 로컬 파일 시스템 기반이다 (클라우드 동기화는 범위 외)
- 자막 프리셋 템플릿 디자인은 사용자 피드백을 받아 반복 개선한다
- edge-tts의 한국어 음성 목록은 최소 5종 이상 제공된다
