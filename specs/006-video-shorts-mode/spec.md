# Feature Specification: 영상 쇼츠 모드

**Feature Branch**: `006-video-shorts-mode`  
**Created**: 2026-04-02  
**Status**: Draft  
**Input**: Phase 6 — 범용 주제 입력 + AI 영상 생성(Seedance) + 이미지 스타일 프리셋 + 파이프라인 분기(manga/video) + UI 업데이트

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 자유 주제로 이미지 쇼츠 영상 만들기 (Priority: P1)

사용자는 블라인드 게시글이 아닌 자유 주제(예: "즐겨 먹던 과자들의 배신", "일상 속 황당한 에피소드")를 입력하여 만화 스타일 Shorts 영상을 만들고 싶다. 주제와 스타일(나레이션/스킷/리뷰), 톤, 부가 설명을 입력하면 기존 이미지 기반 파이프라인으로 영상이 생성된다.

**Why this priority**: 현재 블라인드 전용인 콘텐츠 소스를 범용으로 확장하는 것이 Phase 6의 핵심 목표. 이것이 가능해야 나머지 기능(스타일 선택, AI 영상)이 의미를 가짐.

**Independent Test**: 주제 입력 탭에서 "즐겨 먹던 과자들의 배신"을 나레이션 스타일로 입력 후 생성 → 스크립트 분석 → 이미지 생성 → TTS → 영상 렌더링 완료 확인.

**Acceptance Scenarios**:

1. **Given** 사용자가 웹 UI에 접속한 상태, **When** 주제 입력 탭에서 주제("즐겨 먹던 과자들의 배신"), 스타일(나레이션), 톤(재미), 부가 설명을 입력하고 생성 버튼을 누르면, **Then** AI가 주제를 분석하여 5~8개 씬의 스크립트를 생성하고, 이미지 기반 영상이 만들어진다.
2. **Given** 주제 입력이 완료된 상태, **When** 생성 진행 중일 때, **Then** 각 단계(분석 → 이미지 생성 → TTS → 렌더링)의 진행 상황이 실시간으로 표시된다.
3. **Given** 사용자가 블라인드 스크린샷 입력 탭을 사용할 때, **When** 기존 방식으로 스크린샷을 업로드하면, **Then** 기존과 동일하게 동작한다 (하위 호환성).

---

### User Story 2 - 이미지 스타일 선택 (Priority: P1)

사용자는 영상의 이미지 스타일을 선택하고 싶다. 기존 한국 웹툰 스타일 외에 3D Pixar, 실사풍, 애니메 스타일을 고를 수 있다. 참고 영상 "사물의 잔소리" 채널처럼 3D Pixar 스타일 + 나레이션 조합으로 높은 퀄리티의 영상을 만들 수 있다.

**Why this priority**: 스타일 프리셋은 주제 입력과 함께 즉시 구현 가능하며(API 키 불필요), 영상 다양성을 크게 높여 사용자 만족도에 직접적 영향을 줌.

**Independent Test**: 동일한 주제로 웹툰 스타일과 3D Pixar 스타일을 각각 선택하여 생성 → 두 영상의 이미지 스타일이 명확히 다름을 확인.

**Acceptance Scenarios**:

1. **Given** 사용자가 이미지 모드에서 생성 화면에 있을 때, **When** 이미지 스타일 선택 영역에서 "3D Pixar"를 선택하면, **Then** 생성되는 모든 씬 이미지가 3D Pixar 스타일로 생성된다.
2. **Given** 사용자가 "웹툰" 스타일을 선택한 상태, **When** 영상 생성을 실행하면, **Then** 기존과 동일한 한국 웹툰 스타일 이미지가 생성된다.
3. **Given** 사용자가 "3D Pixar" 또는 "실사풍" 또는 "애니메" 스타일을 선택한 상태, **When** 이미지 생성 시, **Then** 레퍼런스 이미지 기반 편집 대신 순수 생성 방식을 사용한다.

---

### User Story 3 - AI 영상 클립 모드로 쇼츠 만들기 (Priority: P2)

사용자는 정지 이미지 대신 AI가 생성한 3~5초 동영상 클립으로 더 역동적인 Shorts를 만들고 싶다. 비주얼 모드를 "영상 쇼츠"로 전환하면 각 씬마다 AI 영상 클립이 생성된다.

**Why this priority**: AI 영상 생성은 외부 API 키가 필요하며 비용이 높으므로(~$0.05/클립 720p), 기본 기능(주제 입력 + 이미지 스타일)이 안정된 후 추가하는 것이 적절.

**Independent Test**: 영상 쇼츠 모드 선택 → 주제 입력 → 씬별 AI 영상 클립 생성 → TTS + 영상 렌더링 완료 확인.

**Acceptance Scenarios**:

1. **Given** 외부 영상 생성 API 키가 설정된 상태, **When** 사용자가 "영상 쇼츠" 모드를 선택하고 주제를 입력하여 생성하면, **Then** 각 씬마다 3~5초 AI 영상 클립이 생성되고, 최종 영상에 클립이 포함된다.
2. **Given** 영상 쇼츠 모드로 생성 중일 때, **When** 진행 상황을 확인하면, **Then** "씬 2/5: AI 영상 생성 중... (45%)" 형태의 상세 진행률이 표시된다.
3. **Given** 외부 영상 생성 API 키가 설정되지 않은 상태, **When** 사용자가 영상 쇼츠 모드를 선택하면, **Then** API 키 미설정 안내 메시지와 함께 이미지 모드 사용을 권장한다.
4. **Given** AI 영상 생성이 특정 씬에서 실패한 경우, **When** 시스템이 실패를 감지하면, **Then** 해당 씬을 이미지 모드로 폴백 처리하고 사용자에게 알린다.

---

### User Story 4 - 비주얼 모드와 비용 확인 (Priority: P2)

사용자는 이미지 모드와 영상 모드 사이에서 비용 차이를 확인하고 적절한 모드를 선택하고 싶다.

**Why this priority**: 비용 투명성은 사용자 신뢰에 필수적이나, 핵심 기능(주제 입력, 스타일, 영상 생성)이 먼저 동작해야 의미 있음.

**Independent Test**: 모드 토글을 전환하면서 예상 비용이 즉시 변경되는지 확인.

**Acceptance Scenarios**:

1. **Given** 사용자가 생성 화면에 있을 때, **When** 비주얼 모드 토글에서 "이미지 쇼츠"를 선택하면, **Then** 씬당 예상 비용(~$0.005)이 표시된다.
2. **Given** 사용자가 생성 화면에 있을 때, **When** 비주얼 모드 토글에서 "영상 쇼츠"를 선택하면, **Then** 씬당 예상 비용(~$0.05)이 표시된다.
3. **Given** 영상 생성이 완료된 후, **When** 결과 화면을 확인하면, **Then** 실제 사용된 모드와 결과물 요약("이미지 5장" 또는 "영상 5개")이 표시된다.

---

### User Story 5 - 블라인드 모드에서도 영상 클립 사용 (Priority: P3)

기존 블라인드 스크린샷 입력 방식에서도 이미지 대신 AI 영상 클립 모드를 선택할 수 있다.

**Why this priority**: 기존 워크플로우의 확장이므로 신규 기능이 안정된 후 적용하는 것이 안전.

**Independent Test**: 블라인드 스크린샷 업로드 → 영상 쇼츠 모드 선택 → AI 영상 클립 기반 렌더링 완료.

**Acceptance Scenarios**:

1. **Given** 사용자가 블라인드 스크린샷 입력 탭에 있을 때, **When** 비주얼 모드 토글에서 "영상 쇼츠"를 선택하고 스크린샷을 업로드하면, **Then** 기존 분석 + AI 영상 클립 생성 + 렌더링 파이프라인이 실행된다.

---

### Edge Cases

- 사용자가 주제 입력에 빈 값 또는 5자 미만을 입력하면? → 최소 5자 이상 입력 검증, 미달 시 안내 메시지
- AI 영상 생성이 타임아웃(10분 초과)되면? → 해당 씬을 이미지 모드로 자동 폴백, 사용자에게 알림
- 외부 영상 API 잔액 부족 또는 요금 한도 초과 시? → 에러 메시지 표시, 남은 씬은 이미지 모드로 폴백 제안
- 기존 블라인드 모드의 스크린샷 입력과 주제 입력을 동시에 제출하면? → 탭 기반으로 한 번에 하나의 입력 방식만 활성, 동시 제출 불가
- 이미지 스타일 변경 후 이전에 생성된 이미지가 남아있으면? → 새 생성 시 이전 이미지를 덮어씀, 캐시된 이미지 재사용 안 함

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 자유 주제 입력을 받아 Shorts 스크립트를 생성할 수 있어야 한다 (주제, 스타일, 톤, 부가 설명 입력)
- **FR-002**: 시스템은 4가지 이미지 스타일 프리셋(웹툰, 3D Pixar, 실사풍, 애니메)을 제공해야 한다
- **FR-003**: 사용자가 선택한 이미지 스타일에 따라 해당 스타일의 이미지를 생성해야 한다
- **FR-004**: 시스템은 이미지 쇼츠(정지 이미지 기반)와 영상 쇼츠(AI 영상 클립 기반) 두 가지 비주얼 모드를 지원해야 한다
- **FR-005**: 영상 쇼츠 모드에서 각 씬마다 3~5초 AI 영상 클립을 생성해야 한다
- **FR-006**: 시스템은 비주얼 모드별 예상 비용을 사용자에게 표시해야 한다
- **FR-007**: 주제 입력 시 최소 5자 이상의 유효성 검증을 수행해야 한다
- **FR-008**: AI 영상 생성 실패 시 이미지 모드로 자동 폴백해야 한다
- **FR-009**: 생성 진행 상황을 실시간으로 표시해야 한다 (단계명 + 씬 번호 + 비율)
- **FR-010**: 기존 블라인드 스크린샷/직접 입력 워크플로우는 변경 없이 유지되어야 한다 (하위 호환성)
- **FR-011**: 블라인드 입력 모드에서도 비주얼 모드(이미지/영상) 선택이 가능해야 한다
- **FR-012**: 주제 입력 스타일은 나레이션, 스킷, 리뷰 3가지를 지원해야 한다
- **FR-013**: 외부 영상 생성 API 키가 미설정 시 사용자에게 명확한 안내를 제공하고 이미지 모드를 권장해야 한다
- **FR-014**: 결과 화면에서 사용된 비주얼 모드와 결과물 요약(이미지 N장 / 영상 N개)을 표시해야 한다
- **FR-015**: 웹툰 스타일에서만 레퍼런스 이미지 기반 편집을 적용하고, 나머지 스타일은 순수 생성 방식을 사용해야 한다

### Key Entities

- **TopicInput**: 사용자가 입력한 자유 주제 정보. 주제명, 콘텐츠 스타일(나레이션/스킷/리뷰), 톤, 부가 설명을 포함.
- **ImageStylePreset**: 이미지 생성에 사용할 시각적 스타일 프리셋. 웹툰, 3D Pixar, 실사풍, 애니메 4종.
- **VisualMode**: 비주얼 출력 방식 (이미지 쇼츠 또는 영상 쇼츠). 파이프라인 분기의 핵심 분기점.
- **ShortsScript (기존 확장)**: 기존 스크립트 모델에 소스 타입(blind/topic) 구분 추가.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자는 자유 주제 입력 후 3분 이내에 이미지 기반 Shorts 영상을 생성할 수 있다 (기존 블라인드 모드와 동등한 속도)
- **SC-002**: 4가지 이미지 스타일로 생성한 영상이 각각 시각적으로 구분 가능한 스타일을 갖는다 (동일 주제, 다른 스타일 시 다른 결과물)
- **SC-003**: 기존 블라인드 스크린샷 입력 워크플로우가 100% 동일하게 동작한다 (회귀 없음)
- **SC-004**: 영상 쇼츠 모드에서 5개 씬 기준 총 비용이 $0.25(720p) 이하로 유지된다
- **SC-005**: 이미지 쇼츠 모드에서 영상당 총 비용이 $0.03 이하를 유지한다 (기존과 동일)
- **SC-006**: AI 영상 생성 실패 시 이미지 폴백이 자동으로 작동하여 최종 영상 생성 성공률이 95% 이상이다
- **SC-007**: 비주얼 모드 전환과 스타일 선택이 1초 이내에 UI에 반영된다

## Assumptions

- 외부 영상 생성 API의 인터페이스(텍스트→영상 변환, 폴링 기반 상태 확인, 720p/1080p 해상도)는 기획 시점의 사양대로 동작한다.
- 이미지 스타일 프리셋 4종(웹툰, 3D Pixar, 실사풍, 애니메)은 현재 이미지 생성 API의 프롬프트만으로 충분히 차별화 가능하다.
- 주제 입력의 스타일(나레이션/스킷/리뷰) 구분은 AI 분석 프롬프트의 지시 방식 차이로 구현되며, 별도의 AI 모델이 필요하지 않다.
- 외부 영상 API 키는 선택적 설정이며, 미설정 시 영상 쇼츠 모드만 비활성화되고 이미지 쇼츠 모드는 정상 동작한다.
- 영상 쇼츠의 기본 해상도는 비용 효율을 위해 720p로 한다.
- 기존 CLI 파이프라인에서도 visual_mode와 image_style 파라미터를 지원하되, 웹 UI가 우선 구현 대상이다.
