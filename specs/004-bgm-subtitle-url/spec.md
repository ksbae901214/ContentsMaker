# Feature Specification: BGM 자동 삽입 + 자막 줄바꿈 최적화 + URL 콘텐츠 소스 확장

**Feature Branch**: `004-bgm-subtitle-url`
**Created**: 2026-03-25
**Status**: Draft
**Input**: ContentsMaker Phase 4-6 업그레이드 — 영상 품질 향상 및 콘텐츠 소스 확장

## User Scenarios & Testing *(mandatory)*

### User Story 1 - BGM 자동 삽입 (Priority: P1)

사용자가 영상 생성 시 배경음악을 넣을지 선택할 수 있다. 체크박스를 켜면 감정에 맞는 BGM이 영상에 자동 삽입되고, 끄면 TTS 음성만 나온다.

**Why this priority**: 현재 영상이 TTS만 있어 무음 구간이 어색하고 몰입감이 떨어짐. BGM 하나로 영상 퀄리티가 대폭 향상됨.

**Independent Test**: 웹 UI에서 체크박스를 켜고 영상을 생성하면, 완성된 MP4에서 TTS와 BGM이 동시에 재생되는지 확인.

**Acceptance Scenarios**:

1. **Given** 웹 UI 생성 화면, **When** "배경음악 넣기" 체크박스가 ON인 상태에서 영상 생성, **Then** 완성 영상에 TTS + BGM이 함께 재생됨
2. **Given** 웹 UI 생성 화면, **When** "배경음악 넣기" 체크박스가 OFF인 상태에서 영상 생성, **Then** 완성 영상에 TTS만 재생되고 BGM 없음
3. **Given** 감정이 "funny"인 콘텐츠, **When** BGM ON으로 생성, **Then** 경쾌한 분위기의 BGM이 재생됨
4. **Given** 감정이 "angry"인 콘텐츠, **When** BGM ON으로 생성, **Then** 긴장감 있는 분위기의 BGM이 재생됨
5. **Given** BGM ON 영상, **When** 재생, **Then** TTS 음성이 명확히 들리고 BGM은 배경으로 작게 깔림

---

### User Story 2 - 자막 줄바꿈 최적화 (Priority: P1)

영상에 표시되는 텍스트가 화면을 꽉 채우지 않고 문맥에 맞게 줄바꿈되어 가독성이 좋다. 핵심 키워드는 감정에 맞는 색상으로 강조된다.

**Why this priority**: 현재 긴 텍스트가 한 줄로 나와 글씨가 작아지거나 화면을 벗어남. 문맥 단위 줄바꿈으로 가독성 향상.

**Independent Test**: 영상 생성 후 각 씬의 텍스트가 1줄당 15자 내외로 문맥 단위로 줄바꿈되어 있는지 확인.

**Acceptance Scenarios**:

1. **Given** 30자 이상의 씬 텍스트, **When** 영상 생성, **Then** 텍스트가 1줄 15자 내외로 문맥 단위 줄바꿈됨
2. **Given** "회사에서 3년 일했는데 월급이 200도 안 돼요", **When** 분석, **Then** "회사에서 3년 일했는데\n월급이 200도 안 돼요" 형태로 줄바꿈
3. **Given** 감정이 "funny"인 씬, **When** 영상 렌더링, **Then** 핵심 키워드가 노란색으로 강조됨
4. **Given** 감정이 "angry"인 씬, **When** 영상 렌더링, **Then** 핵심 키워드가 빨간색으로 강조됨
5. **Given** 줄바꿈된 텍스트, **When** 화면에 표시, **Then** 화면 가로 80% 이내에 들어와 양쪽 여백이 충분함

---

### User Story 3 - URL 입력으로 콘텐츠 생성 (Priority: P2)

사용자가 웹 UI에서 게시글 URL을 입력하면, 시스템이 자동으로 해당 사이트에서 제목/본문/댓글을 추출하여 영상을 생성한다.

**Why this priority**: 현재는 스크린샷을 찍거나 직접 입력해야 함. URL 입력만으로 자동 추출되면 작업 시간이 대폭 줄어듦.

**Independent Test**: 디시인사이드 게시글 URL을 입력하면 제목, 본문, 댓글이 추출되어 영상이 생성되는지 확인.

**Acceptance Scenarios**:

1. **Given** 웹 UI의 "URL 입력" 탭, **When** 디시인사이드 게시글 URL 입력 후 생성 클릭, **Then** 제목/본문/댓글이 자동 추출되어 영상 생성 시작
2. **Given** 네이트판 게시글 URL, **When** URL 입력 후 생성, **Then** 제목/본문/댓글이 추출되어 영상 생성됨
3. **Given** 네이버 카페 게시글 URL, **When** URL 입력 후 생성, **Then** 제목/본문/댓글이 추출되어 영상 생성됨
4. **Given** 지원하지 않는 사이트 URL, **When** URL 입력 후 생성, **Then** "지원하지 않는 사이트입니다" 오류 메시지 표시
5. **Given** 비공개/삭제된 게시글 URL, **When** URL 입력 후 생성, **Then** "게시글을 찾을 수 없습니다" 오류 메시지 표시
6. **Given** CLI 환경, **When** `python3 -m src.main url <URL>` 실행, **Then** URL에서 추출 후 영상 생성

---

### Edge Cases

- BGM 파일이 누락된 경우 → BGM 없이 TTS만으로 영상 생성 (경고 표시)
- 텍스트가 5자 미만으로 매우 짧은 경우 → 줄바꿈 없이 한 줄로 표시
- 네이버 카페 게시글이 로그인 필요한 경우 → "로그인이 필요한 게시글입니다" 오류 표시
- URL이 게시글이 아닌 목록 페이지인 경우 → "게시글 URL을 입력해주세요" 오류 표시
- 댓글이 없는 게시글 → 댓글 없이 본문만으로 영상 생성
- 이미지만 있는 게시글 (텍스트 없음) → "텍스트 콘텐츠가 부족합니다" 오류 표시

## Requirements *(mandatory)*

### Functional Requirements

#### BGM
- **FR-001**: 웹 UI에 "배경음악 넣기" 체크박스가 존재해야 하며, 기본값은 ON
- **FR-002**: 시스템은 감정 타입(funny/touching/angry/relatable)별로 서로 다른 BGM을 선택해야 함
- **FR-003**: BGM은 영상 전체에 걸쳐 재생되며, TTS 음성을 방해하지 않는 볼륨이어야 함
- **FR-004**: CLI에서 `--no-bgm` 옵션으로 BGM을 비활성화할 수 있어야 함

#### 자막 줄바꿈
- **FR-005**: AI 분석 단계에서 씬 텍스트에 문맥 기반 줄바꿈을 자동 삽입해야 함
- **FR-006**: 1줄당 최대 15자 내외로, 조사/쉼표/의미 단위 뒤에서 줄바꿈해야 함
- **FR-007**: 감정별 핵심 키워드 하이라이트 색상이 적용되어야 함 (funny=노랑, touching=분홍, angry=빨강, relatable=하늘색)

#### URL 콘텐츠 소스
- **FR-008**: 웹 UI에 "URL 입력" 탭이 추가되어야 함
- **FR-009**: 시스템은 URL에서 사이트를 자동 감지하고, 지원 사이트인 경우 제목/본문/댓글을 추출해야 함
- **FR-010**: 디시인사이드, 네이트판, 네이버 카페 게시글을 지원해야 함
- **FR-011**: 지원하지 않는 사이트이거나 접근 불가 시 사용자에게 명확한 오류 메시지를 표시해야 함
- **FR-012**: CLI에서 `python3 -m src.main url <URL>` 명령으로 URL 기반 영상 생성이 가능해야 함

### Key Entities

- **BGM 파일**: 감정 타입과 매핑되는 오디오 파일. 로열티프리 라이선스. 4종.
- **줄바꿈된 텍스트**: 문맥 단위로 `\n`이 삽입된 씬 표시 텍스트. 1줄 15자 내외.
- **URL 파서**: 사이트별로 게시글 데이터를 추출하는 모듈. URL → BlindPost 변환. 3개 사이트 지원.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: BGM ON 영상과 OFF 영상이 모두 정상 생성되어 체크박스 옵션이 동작함
- **SC-002**: 모든 씬 텍스트가 1줄 15자 내외로 줄바꿈되어 화면 가로 80% 이내에 표시됨
- **SC-003**: 감정별 키워드 하이라이트 색상이 영상에서 육안으로 구분됨
- **SC-004**: 디시인사이드, 네이트판, 네이버 카페 URL 입력 시 영상 생성 성공
- **SC-005**: 지원 사이트 3곳에서 각 1개 이상의 게시글 URL로 영상 생성 테스트 통과
- **SC-006**: BGM 재생 시 TTS 음성이 명확히 청취 가능함

## Assumptions

- BGM 파일은 Pixabay Music 등 로열티프리 소스에서 수동으로 다운로드하여 `data/bgm/` 폴더에 배치
- BGM 볼륨은 TTS 대비 15-20% 수준 (고정값, 사용자 조절 없음)
- 네이버 카페는 공개 게시글만 지원 (로그인 필요 게시글은 미지원)
- 키워드 하이라이트는 AI 분석 단계에서 자동 선정 (사용자 수동 지정 없음)
- URL 파싱은 Playwright를 사용하여 동적 렌더링된 페이지에서 추출
