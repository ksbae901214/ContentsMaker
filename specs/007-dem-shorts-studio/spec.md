# Feature Specification: 정당 관점 기반 정치 쇼츠 반자동 제작 시스템 (Party-Shorts Studio)

**Feature Branch**: `007-dem-shorts-studio`
**Created**: 2026-04-16
**Status**: Draft (v2 — 2026-04-20 perspective axis 도입)
**Input**: User description: "정치쇼츠_기획패키지.md 기획안으로 새로운 영상 생성 기능을 추가. 기존 정치해설 모드(006)는 건드리지 않고 별도로 개발"

## Overview

NATV(국회방송) 공개 영상을 소스로, 운영자가 선택한 **정당 관점(`party_perspective`)** 에 맞춰 친화형·팩트 기반 정치 쇼츠를 반자동으로 제작하는 별도 워크플로우. 관점은 `dem`(민주당·조국혁신당 친화) 또는 `ppp`(국민의힘 친화) 중 선택하며, 시스템 자체는 특정 정당 편향을 강제하지 않고 운영자 선택에 따라 톤·pinned 인물·가드레일 방향이 대칭적으로 달라진다. 운영자(프로젝트 오너) 1인이 검수 게이트를 거쳐 YouTube에 하루 1개씩 업로드하여 6개월 내 YouTube 파트너 프로그램 수익화 승인을 목표로 한다.

기존 006 정치해설 모드(국회의원 단발 영상 → AI 해설 교차편집)는 그대로 보존하고, 본 기능은 별도 모드(`mode=dem_shorts`, 후속 리팩토링에서 `party_shorts`로 리네임 예정)로 추가한다.

**perspective ↔ channel 고정 매핑**: 각 운영 YouTube 채널은 하나의 perspective와 1:1로 고정된다. 현재 프로덕션 채널(`@국회직캠-d6r`)은 **`ppp` perspective 전용**으로 운영하며, `dem` perspective는 로컬 렌더 테스트용으로만 사용한다(업로드 불가). 추후 dem 관점 별도 채널 개설 시 `perspectives` 테이블에 `channel_id` 매핑을 추가한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 신규 NATV 영상에서 민주당 세션 우선순위 확인 (Priority: P1)

운영자가 매일 아침 대시보드를 열면, 지난 24시간 동안 NATV 채널에 새로 업로드된 영상들이 "민주당 점유도" 기준으로 순위화되어 표시된다. 운영자는 점수가 높은 영상부터 클릭해 편집에 착수한다.

**Why this priority**: 이 기능이 없으면 운영자가 매일 수시간씩 NATV를 뒤져야 한다. 소스 탐색 자동화는 1인 운영 지속가능성의 전제 조건이며, 전체 파이프라인의 입구이기 때문에 가장 먼저 동작해야 한다.

**Independent Test**: 대시보드 진입 → 최근 24시간 신규 NATV 영상 목록이 민주당 우선순위 점수와 함께 표시됨을 확인. 새 영상 1개를 수동 투입한 뒤 30분 이내에 자동 감지·점수화되는지 확인.

**Acceptance Scenarios**:

1. **Given** NATV 채널에 지난 24시간 동안 신규 영상 3개가 업로드된 상태, **When** 운영자가 대시보드에 접속, **Then** 3개 영상 모두 민주당 점유도 점수와 함께 내림차순 정렬되어 표시된다.
2. **Given** 민주당 인물이 전혀 등장하지 않는 영상이 신규 수집된 상태, **When** 점수 계산이 완료됨, **Then** 해당 영상은 "자동 제외" 상태로 표시되어 기본 목록에서 숨겨진다.
3. **Given** 영상 길이가 6시간을 초과하는 본회의 영상이 감지된 상태, **When** 다운로드 큐에 진입, **Then** "길이 초과" 사유로 자동 제외되고 알림이 기록된다.

### User Story 2 — Whitelist 정치인 발언 구간 자동 하이라이트 (Priority: P1)

운영자가 우선순위 영상을 선택하면, 시스템이 Whitelist에 등록된 정치인(이재명·조국·정청래 + 여성·청년 상위 20명)의 발언 구간을 타임라인에 자동 하이라이트한다. 운영자는 하이라이트된 구간 중 추천 점수가 높은 1개를 선택해 쇼츠로 자를 수 있다.

**Why this priority**: 발언자 자동 식별은 전체 작업 시간을 2시간 → 30분으로 단축하는 핵심 자산이다. 이 기능 없이는 1일 1개 업로드 목표가 불가능하며, 여성·청년 정치인 차별화(프로젝트의 유일한 해자)도 실현되지 않는다.

**Independent Test**: 이재명·조국·정청래가 각각 발언한 2시간짜리 테스트 영상을 투입 → 3명의 발언 구간이 색상별로 타임라인에 표시되고, 구간별 추천 점수가 계산됨을 확인.

**Acceptance Scenarios**:

1. **Given** Whitelist에 이재명·조국·정청래가 등록된 상태에서 3명 모두 등장하는 영상을 열었을 때, **When** 자동 식별이 완료됨, **Then** 타임라인에 3개 색상으로 발언 구간이 표시되고 각 구간 옆에 인물 이름과 신뢰도(confidence)가 표시된다.
2. **Given** Whitelist에 없는 타당 인물이 발언 중인 경우, **When** 구간 식별됨, **Then** "(타당)" 또는 "(미식별)" 태그로 표시되고 기본적으로 하이라이트에서 제외된다.
3. **Given** 여성/청년 랭킹 상위 20명 배치가 완료된 상태, **When** 랭킹 신규 진입자가 영상에 등장, **Then** "신규" 태그와 함께 하이라이트된다.

### User Story 3 — 해설 자막 작성 + TTS + 컴플라이언스 검수 → 쇼츠 렌더 (Priority: P1)

운영자가 자른 60초 이하 원본 구간 위에 AI 추천 해설 자막 3개 후보 중 하나를 고르거나 직접 편집한다. TTS 나레이션을 선택적으로 추가하고, 업로드 전 10개 항목 컴플라이언스 게이트를 통과해야 최종 쇼츠 MP4가 생성된다.

**Why this priority**: 본인 해설 자막 50자 이상 + 원본 50% 이하는 YouTube 수익화 심사 통과의 핵심 조건이다. 이 게이트가 없거나 우회 가능하면 R-01(수익화 거부) Critical 리스크가 실현되며, 전체 프로젝트 목표가 무너진다.

**Independent Test**: 테스트 영상 구간 + 해설 자막 50자 + 팩트 출처 2개를 입력 → 10개 게이트 중 자동 8개 통과 + 수동 2개 체크 → 렌더링 버튼 활성화됨을 확인. 해설 자막 50자 미만으로 수정 시 버튼이 즉시 비활성화되는지 확인.

**Acceptance Scenarios**:

1. **Given** 해설 자막이 50자 이상이고 원본 비율이 50% 이하이며 출처 2개가 첨부된 상태, **When** 운영자가 업로드 전 검수를 누름, **Then** 자동 8개 항목 모두 통과로 표시되고 수동 체크 2개가 활성화된다.
2. **Given** 해설 자막이 49자 이하인 상태, **When** 게이트 검증 실행, **Then** "본인 해설 자막 분량 미달" 사유로 업로드 버튼이 비활성화되고 현재 글자 수가 빨간색으로 표시된다.
3. **Given** 명예훼손 리스크 스코어가 61점 이상인 상태, **When** 게이트 검증 실행, **Then** "위험" 등급으로 강제 차단되고 문제 문장이 강조되며 대안 문구가 제시된다.

### User Story 4 — 선거기간 자동 감지 + 중립 모드 강제 (Priority: P2)

대선(D-180) 또는 총선(D-120) 기간이 되면 시스템이 자동으로 경고 배너를 띄우고, 편향 게이트 임계값을 상향하며, "중립 모드"를 강제 활성화한다. 중립 모드에서는 친화형 톤앤매너가 "정책 설명 중심"으로 전환된다.

**Why this priority**: 공직선거법 위반(R-03 Critical)은 채널 강제 정지·형사 입건 리스크다. 기본 기능은 아니지만 선거기간에만 반드시 동작해야 하므로 P2. 선거기간 외에는 트리거되지 않아 평시 운영에 영향이 없다.

**Independent Test**: 시스템 시각을 선거일 D-181 → D-179로 변경 → 경고 배너가 자동 표시되고 편향 게이트 임계값이 자동 상향됨을 확인.

**Acceptance Scenarios**:

1. **Given** 대선일 D-200 상태, **When** D-180 경계를 넘음, **Then** 대시보드 상단에 "선거기간 시작 — 중립 모드 활성화" 배너가 표시되고 일일 알림이 발송된다.
2. **Given** 선거기간 중 해설 자막에 특정 후보 우호 표현이 감지됨, **When** 저장 시도, **Then** 중립 모드 위반으로 경고되고 대안 문구가 제안된다.
3. **Given** 선거일이 지난 시점, **When** 자동 체크 실행, **Then** 배너가 자동 해제되고 기존 친화형 모드로 복귀한다.

---

### User Story 5 — 월간 편향 밸런스 리포트 조회 (Priority: P3)

매월 1일 시스템이 지난달 업로드한 쇼츠들의 인물별·정당별 언급 비율, 사용 템플릿 분포, 리스크 스코어 추이를 집계한 리포트를 생성한다. 운영자는 리포트를 보고 다음달 콘텐츠 다양성을 조정한다.

**Why this priority**: 특정 인물·템플릿 편중은 중장기적으로 채널 피로도와 팬덤 분열(R-16)을 초래한다. 그러나 즉각적인 차단 요소는 아니므로 운영 안정화 이후 추가해도 되는 P3.

**Independent Test**: 지난달 30개 쇼츠 업로드 이력 시뮬레이션 → 리포트 페이지에 인물별 점유율·정당 비율·템플릿 사용 빈도가 차트로 표시됨을 확인.

**Acceptance Scenarios**:

1. **Given** 지난달 이재명 쇼츠 15개·기타 15개 업로드한 이력, **When** 월간 리포트 생성, **Then** "이재명 50%"로 명시되고 권장 상한(30%) 초과 경고가 표시된다.
2. **Given** 타 정당 인물 언급 비율이 10% 미만인 이력, **When** 리포트 조회, **Then** "객관성 리스크 — 타당 언급 부족" 권고가 포함된다.

### Edge Cases

- **NATV 채널 API 쿼터 초과**: 폴링 큐에 대기 상태로 남고 다음 주기에 재시도. 운영자에게 경고 알림.
- **Whisper STT 실패 (음질 불량)**: 해당 영상에 "STT_FAIL" 플래그 → UI에서 수동 자막 편집 모드 안내.
- **발언자 식별 신뢰도 0.7 미만**: 하이라이트 없이 "(미식별)" 표시, 수동 선택 가능.
- **디스크 여유 50GB 미만**: 신규 영상 수집 일시 중단 + 운영자 알림 → 오래된 원본 아카이브를 콜드 스토리지로 이동 후 재개.
- **동일 정치인 연속 3회 업로드**: 경고 표시되지만 차단되지 않음(운영자 판단).
- **Whitelist 비활성화(OFF) 인물이 영상에 등장**: 식별은 계속되지만 기본 하이라이트에서 제외. 통계에는 집계.
- **Blocked 등급 인물 감지**: 해당 발언 구간이 자동 회색 처리되고 쇼츠 대상에서 제외. 사유 로그 남김.
- **NATV 비회기 공백 (국회 휴회 기간)**: 신규 영상 0건이어도 오류 아님. 대시보드에 "휴회 기간" 표시.
- **소스 영상이 게시 후 삭제됨**: 다운로드 아카이브 원본 유지, 레코드에 "원본 삭제" 태그.
- **Whitelist가 0명인 상태**: 쇼츠 제작 시작 불가, UI에서 "최소 1명 등록 필요" 안내.

## Requirements *(mandatory)*

### Functional Requirements

#### 영상 소스 수집 (Source Discovery)

- **FR-001**: 시스템은 NATV YouTube 채널을 기본 30분 주기로 폴링하여 신규 영상을 자동 감지해야 한다 (폴링 주기는 설정 변경 가능).
- **FR-002**: 시스템은 감지된 신규 영상의 원본 파일을 로컬 아카이브에 다운로드하고, 영상 길이 6시간 초과 건은 자동 제외해야 한다.
- **FR-003**: 시스템은 영상 제목·설명을 파싱하여 세션 타입(본회의/상임위원회/국정감사/청문회/기자회견/기타)을 자동 분류해야 한다.
- **FR-004**: 시스템은 다운로드된 영상 각각에 대해 **perspective 점유도 점수**(`perspective_score`, 0~100)를 계산하여 대시보드에 내림차순 정렬로 표시해야 한다. 활성 perspective의 Whitelist 인물 등장 빈도·TOP 인물 가중치 기반. (기존 `dem_score`는 `perspective='dem'`일 때의 별칭으로 하위호환 유지)
- **FR-005**: 활성 perspective의 Whitelist 인물이 0명 감지된 영상은 기본 목록에서 "자동 제외"로 숨기며, 운영자가 필터를 해제하면 조회 가능해야 한다.

#### 타겟 정치인 관리 (Target Management)

- **FR-006**: 시스템은 Whitelist에 perspective별 pinned 시드 인물을 기본 등록해야 한다.
  - `dem` perspective: 이재명·조국·정청래 (3명)
  - `ppp` perspective: 한동훈·김기현·권성동·추경호·나경원·오세훈 (6명)
  - 각 `Politician` 엔티티는 `affiliation_perspective` 필드를 가진다.
- **FR-007**: 운영자는 Whitelist에 정치인을 고정(Pinned) / 자동(Auto) / 제외(Blocked) 등급으로 등록·수정·삭제할 수 있어야 한다.
- **FR-008**: 시스템은 여성·청년 카테고리 정치인의 외부 언급량(뉴스 기사 수, 검색량 추이, 관련 영상 조회수, 소셜 언급, 백과 페이지뷰)을 주 1회 자동 집계하여 인기도 점수(0~100)를 산출해야 한다.
- **FR-009**: 주간 랭킹 상위 20명은 Whitelist "자동(Auto)" 등급에 자동 등록되며, 21위 이하는 "대기(Pending)" → 2주 연속 대기 시 제외되어야 한다.
- **FR-010**: 각 Whitelist 인물은 사진, 소속, 대표 이력, 톤앤매너 가이드(콘텐츠 생성 시 해설 톤 참조용)를 저장할 수 있어야 한다.
- **FR-011**: 운영자는 "연합 대상 토글"을 통해 활성 perspective의 연합 진영 인물 Whitelist 포함 여부를 전체 ON/OFF 할 수 있어야 한다. (dem 관점: 조국혁신당 등 범민주, ppp 관점: 개혁신당 등 범보수)

#### 발언자 식별 (Speaker Identification)

- **FR-012**: 시스템은 다운로드된 영상에 대해 STT(음성 전사)를 자동 실행하고 시간별 자막 데이터를 저장해야 한다.
- **FR-013**: 시스템은 STT 결과에 화자 분리(diarization)를 적용하고, 자막 호명 패턴 + 영상 OCR(이름표) + 출석자 명단 대조 3가지 단서를 결합하여 각 발언 구간의 화자를 식별해야 한다.
- **FR-014**: 식별 신뢰도(confidence) 0.7 이상의 구간만 "확정"으로 표시하고, 나머지는 "(미식별)"로 표시해야 한다.
- **FR-015**: 시스템은 Whitelist 매칭 결과에 따라 인물별 색상으로 타임라인에 발언 구간을 하이라이트해야 한다.
- **FR-016**: 시스템은 각 발언 구간에 대해 쇼츠 추천 점수(발언 길이 적정성, 감정 강도, 이슈 키워드, 단독 발언 가중치, 욕설 감점)를 계산하여 정렬해야 한다.

#### 편집 및 해설 작성 (Editing & Commentary)

- **FR-017**: 운영자는 타임스탬프 기반 네비게이션 UI에서 재생·탐색하며 임의 구간을 시작~종료로 자를 수 있어야 한다.
- **FR-018**: 시스템은 자른 구간을 9:16 세로 포맷(1080×1920)으로 자동 변환하되 최대 60초 이내여야 한다.
- **FR-019**: 운영자는 자른 영상 위에 해설 자막을 타임라인 기반으로 삽입·수정·삭제할 수 있어야 한다.
- **FR-020**: 시스템은 영상의 STT 전문 + 등장 인물 + 톤앤매너 가이드를 기반으로 AI 해설 자막 후보 3개(각 15자 이내, 팩트 기반, 단정 표현 금지)를 자동 생성해야 한다.
- **FR-021**: 시스템은 인물별 자막 스타일 프리셋(이재명/정청래/청년 정치인/핫이슈/일반 해설 5종)을 제공해야 한다.
- **FR-022**: 운영자는 TTS 나레이션 보이스 프리셋(남-박력/남-안정/여-차분/여-젊음 4종) 중 선택해 해설에 추가할 수 있어야 한다.
- **FR-023**: 시스템은 원본 음성을 -12dB로 자동 다운하고 TTS 나레이션을 0dB로 오버레이해야 한다.
- **FR-024**: 시스템은 해설 자막 누적 글자 수를 실시간 표시하고, 50자 미만이면 경고를 표시해야 한다.

#### 컴플라이언스 게이트 (Compliance Gate)

- **FR-025**: 시스템은 업로드 직전 다음 11개 항목을 검증하는 강제 게이트를 제공해야 한다. **어떤 운영자 조작으로도 우회할 수 없어야 한다** (item 9·10·11은 운영자 서명으로 override 허용, 감사 로그 필수).
  1. 본인 해설 자막 50자 이상 (자동, 차단)
  2. 원본 비율 ≤ 50%, 해설 비율 ≥ 30% (자동, 차단)
  3. 영상 길이 60초 이내 (자동, 차단)
  4. 설명란에 "NATV 국회방송" 출처 표시 (자동, 차단)
  5. 편향·혐오 가드레일 통과 (자동, 경고) — 활성 perspective 기준
  6. 동일 템플릿 최근 3회 연속 사용 아님 (자동, 경고)
  7. 활성 perspective의 Whitelist 인물 최소 1명 포함 (자동, 차단)
  8. 선거기간 가드 통과 (자동, 차단) — perspective 무관 대칭 적용
  9. 팩트 검증 완료 (운영자 수동 체크, 차단)
  10. 명예훼손 요소 없음 (운영자 수동 체크, 차단)
  11. **대칭성(Symmetry) 가드** — 동일 해설 스크립트를 반대 perspective로 평가했을 때 risk_score 차이가 20점을 초과하면 경고(첫 30일) 후 차단으로 승격. 운영자 서명 override 허용. (우리편 과도 옹호·반대편 비방 탐지)
- **FR-026**: 시스템은 혐오·편향·허위사실·명예훼손 4개 카테고리별 리스크 스코어(0~100)를 계산하고, 61점 이상이면 업로드를 강제 차단해야 한다. 가드레일 키워드 사전은 perspective 무관 대칭 작동(현행) + perspective별 `against_us`/`about_them` 서브사전 병합.
- **FR-027**: 시스템은 해설 자막·TTS 원고·메타데이터에 대해 키워드 룰(빠름) + LLM 분류(느림·정확) 2단계 검출 엔진을 적용해야 한다. LLM 프롬프트는 활성 perspective를 인지하여 "우리편 과도 옹호 / 반대편 비방" 양방향을 감지한다.
- **FR-028**: 시스템은 운영자가 수정·무시한 가드레일 건을 이력으로 누적하고, 월 1회 패턴을 재학습해야 한다. 학습은 perspective별로 분리되며 교차 오염이 없어야 한다.
- **FR-029**: 시스템은 업로드될 설명란에 NATV 출처 + 팩트 뉴스 링크 최소 2개가 포함되었는지 검증해야 한다.

#### 선거법 가드 (Election Law Guard)

- **FR-030**: 시스템은 대선일 D-180 / 총선일 D-120 경계를 자동 감지하여 선거기간 진입 시 대시보드에 경고 배너를 표시해야 한다.
- **FR-031**: 선거기간 중에는 중립 모드가 강제되며, 편향 리스크 스코어 임계값이 30점 이하로 하향되어야 한다.
- **FR-032**: 중립 모드에서 AI 해설 생성은 "특정 후보·정당 우호 표현 금지, 정책 설명 중심" 프롬프트로 전환되어야 한다.

#### 렌더링 및 업로드 (Render & Upload)

- **FR-033**: 시스템은 원본 구간 + 해설 자막 + TTS + BGM + 선택적 인트로/아웃트로를 합성한 최종 MP4를 H.264 / 1080×1920 / 30fps / AAC 128kbps 포맷으로 렌더링해야 한다.
- **FR-034**: 시스템은 자막만 변경된 재렌더링 요청에 대해 스마트 캐싱을 적용하여 전체 렌더링 시간의 1/3 이내로 완료해야 한다.
- **FR-035**: 시스템은 저작권 프리 BGM 라이브러리만 사용해야 하며, 업로드 시 라이선스 메타데이터를 검증해야 한다.
- **FR-036**: 시스템은 컴플라이언스 게이트를 통과한 쇼츠를 YouTube에 예약 발행(시간·제목·설명·태그 자동 입력)할 수 있어야 한다.
- **FR-037**: 업로드 전 운영자의 최종 "업로드 확정" 버튼 클릭이 필수이며, 완전 자동 업로드는 제공하지 않아야 한다.

#### 운영 및 모니터링 (Operations)

- **FR-038**: 시스템은 월 1회 자동으로 편향 밸런스 리포트(인물별 점유율, 템플릿 사용 빈도, 리스크 스코어 추이)를 **활성 perspective별로 분리 생성**해야 한다. 단일 perspective 운영 채널(예: `ppp` only)의 경우, 내부 다양성(예: 한동훈 vs 김기현 등장 비율) 관점에서 분석되어야 한다.
- **FR-039**: 운영자는 자막·BGM·인트로 프리셋을 템플릿 관리 UI에서 등록·편집·삭제할 수 있어야 한다.
- **FR-040**: 시스템은 원본 영상 파일, 편집 이력, 가드레일 통과 기록, 업로드 이력을 최소 1년간 보존해야 한다 (명예훼손 피소 대응 증거).

#### 기존 기능 분리 (Isolation from Existing)

- **FR-041**: 본 기능은 기존 006 정치해설 모드(`mode=political`)와 **완전히 독립된 별도 워크플로우**(`mode=dem_shorts`)로 동작해야 한다. 기존 정치해설 UI·API·데이터 모델은 변경되어서는 안 된다.
- **FR-042**: 본 기능은 기존 ContentsMaker의 TTS·STT·썸네일 생성 모듈을 재활용하되, 원본 호출부를 수정하지 않고 확장 인터페이스로 통합해야 한다.

### Key Entities

- **Perspective (정당 관점)**: 시스템이 지원하는 관점 축. 속성: id(`dem`|`ppp`), label, channel_id(1:1 고정, nullable — 현재 dem은 NULL로 업로드 비활성), is_active. (신규 엔티티, 2026-04-20)
- **SourceVideo (소스 영상)**: NATV에서 수집된 원본 영상. 속성: video_id, 제목, 설명, 게시일, 길이, 세션 타입, 다운로드 경로, STT 상태, **perspective별 점유도 점수(`perspective_score`)**, target_perspective, 상태(신규/처리중/완료/제외). (`dem_score`는 `perspective='dem'`일 때의 레거시 별칭)
- **Politician (정치인)**: Whitelist 관리 대상. 속성: 이름, 소속, 직책, 사진, 대표 이력, 톤앤매너 가이드, 등급(Pinned/Auto/Pending/Blocked), 카테고리(고정/여성/청년/연합), **affiliation_perspective (`dem`|`ppp`)**, 활성 여부, 주간 랭킹 점수.
- **SpeechSegment (발언 구간)**: 영상 내 특정 시작~종료 시각의 발언 단위. 속성: source_video_id, start/end, 화자(Politician 참조), 신뢰도, STT 텍스트, 쇼츠 추천 점수, 감정 강도, 이슈 키워드.
- **ShortsDraft (쇼츠 초안)**: 운영자가 발언 구간에서 제작 중인 쇼츠. 속성: 기반 SpeechSegment, 자른 구간, 해설 자막 블록 리스트, TTS 설정, 자막 스타일 프리셋, BGM 선택, 리스크 스코어, 게이트 통과 여부, 상태(초안/검수중/통과/실패).
- **ComplianceGateResult (게이트 결과)**: 특정 ShortsDraft의 10개 항목 검증 이력. 속성: 각 항목별 통과/실패/경고 상태, 실패 사유, 타임스탬프, 운영자 수동 체크 서명.
- **WeeklyRanking (주간 랭킹)**: 여성·청년 정치인 인기도 집계. 속성: 주차, 순위, 인물, 점수, 전주 대비 증감, 태그(신규/급상승/대기).
- **UploadedShorts (업로드된 쇼츠)**: 게이트 통과 후 YouTube에 발행된 최종 쇼츠. 속성: 쇼츠 draft 참조, 최종 MP4 경로, YouTube video_id, 업로드 시각, 제목·설명·태그, 팩트 링크, 조회수·수익 메트릭.
- **BiasReport (편향 리포트)**: 월별 집계 결과. 속성: 월, 인물별 점유율, 정당별 비율, 템플릿 사용 빈도, 평균 리스크 스코어, 권고 메시지.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 운영자는 NATV 신규 영상 감지부터 YouTube 업로드 완료까지 영상 1개를 **30분 이내**에 완료할 수 있다.
- **SC-002**: 시스템은 신규 업로드된 NATV 영상을 **30분 이내**에 자동 감지하여 대시보드에 표시한다.
- **SC-003**: Whitelist 인물 발언 구간 자동 식별 정확도가 **80% 이상**이다 (운영 시작 시점, 신뢰도 0.7 기준).
- **SC-004**: 활성 perspective 우선순위 점수가 높은 상위 5개 영상에 대해, 운영자가 실제 선택하는 비율이 **70% 이상**이다 (점수의 실효성 지표).
- **SC-005**: 컴플라이언스 게이트는 **100% 우회 불가**하며, 11개 항목 중 하나라도 실패하면 업로드 버튼이 비활성화된다 (item 9·10·11은 운영자 서명 override 허용, 감사 로그 필수).
- **SC-006**: 업로드된 모든 쇼츠는 본인 해설 자막 **평균 70자 이상**, 원본 구간 비율 **평균 50% 이하**를 유지한다.
- **SC-007**: 선거기간 경계(D-180/D-120) 진입 시 **24시간 이내**에 대시보드 경고 배너가 자동 표시된다.
- **SC-008**: 시스템은 월 30개 쇼츠 생산 기준으로 렌더링 실패율 **5% 이하**를 유지한다.
- **SC-009**: 프로젝트 시작 후 **6개월 시점**에 YouTube 파트너 프로그램 신청 요건(구독자 500명 + 시청 시간)을 충족한다.
- **SC-010**: 운영 개시 후 **12개월간** 명예훼손 피소 / 선거법 위반 / YouTube 계정 정지 이슈가 **0건**이다.
- **SC-011**: 월간 편향 밸런스 리포트에서 **활성 perspective의 특정 인물 비중이 30% 이하**로 유지된다. 각 perspective의 pinned 시드 인물 합계는 60% 이하. (dem 관점: 이재명·조국·정청래 3인 합계 ≤60%. ppp 관점: 한동훈·김기현·권성동·추경호·나경원·오세훈 6인 중 임의 3인 합계 ≤60%)
- **SC-012**: 여성·청년 정치인 카테고리 쇼츠가 전체 업로드의 **40% 이상**을 차지한다 (차별화 포지셔닝 지표).
- **SC-013 (신규, 2026-04-20)**: Symmetry 게이트(FR-025 item 11)는 첫 30일 경고 모드로 동작하고 31일째부터 차단 모드로 자동 승격된다. 승격 후 fail 발생 시 운영자 서명 override 비율은 월 10% 이하여야 한다. (과도한 override는 정책 재검토 신호)
- **SC-014 (신규, 2026-04-20)**: perspective ↔ channel_id 하드블록(FR-036 연관)은 **100% 우회 불가**. 잘못된 perspective 영상을 업로드 시도하면 시스템이 TypeError로 즉시 차단한다.

## Assumptions

- **프로젝트 오너 1인 운영**: 기획·개발·검수·업로드를 1인이 담당. 주 5일 운영 + 3일치 예약 발행 버퍼 + 월 최대 4일 "스킵 데이" 허용.
- **기존 ContentsMaker 자산 재활용**: TTS·STT·썸네일/타이틀 생성 모듈은 기존 구현을 그대로 재사용하며 본 기능은 확장 레이어로만 동작.
- **외부 라이브러리 성숙도 의존**: 화자 분리, STT, 영상 처리, 다운로드는 성숙한 공개 라이브러리를 활용한다고 가정.
- **NATV 저작권**: NATV(국회방송) 영상은 공공 목적 재사용이 가능하되, 설명란에 출처 표시가 필수 조건이라고 가정. 법률 자문 후 최종 확인.
- **LLM 사용 비용**: 쇼츠 1개당 약 100원 (가드레일 + 해설 후보 + 타이틀 생성 합산). 월 30개 기준 3,000원 예산 내.
- **스토리지**: 외장 SSD 2TB 준비. 6시간 본회의 영상 원본은 평균 5~10GB이므로 월 약 300GB 수집 → 3개월 주기로 콜드 스토리지 이동.
- **선거법 기준**: 대선 D-180, 총선 D-120을 공직선거법 기준일로 가정. 선관위 가이드라인 변경 시 설정 갱신.
- **Whitelist 초기 등록**: 이재명·조국·정청래 3명을 "고정(Pinned)" 등급으로 시스템 설치 시 자동 등록.
- **법률 자문 전제**: 프로젝트 시작 전 정치·언론 전문 변호사 1회 자문(운영자 별도 진행)을 받은 것으로 가정. 명예훼손 임계값·단정 표현 블랙리스트는 자문 결과 반영.
- **범위 제외**: 완전 자동 업로드, 다국어, 멤버십·슈퍼챗, 타 플랫폼 동시 배포, 공격형 콘텐츠, BGM 라이선스 계약 대행.

## Dependencies

- **기존 006 정치해설 모드**: 본 기능과 완전히 독립되어야 하며, 006 코드 경로에 변경을 가하지 않아야 한다.
- **외부 데이터 소스**:
  - NATV YouTube 채널 메타데이터·영상 (일일 쿼터 내 폴링)
  - 뉴스·검색 트렌드 데이터 (여성/청년 랭킹 계산)
  - LLM 서비스 (해설 후보 생성 + 편향 분류)
- **하드웨어**: GPU 가용 시 렌더링 가속(60초 쇼츠 3분 이내), CPU만 사용 시 최대 8분 허용.
- **법률/규정**: 저작권법, YouTube 커뮤니티 가이드, 공직선거법, 정보통신망법(명예훼손), 정치자금법.
- **운영자 자원**: 외장 SSD 2TB, 저작권 프리 BGM 라이브러리, YouTube 채널 2개(메인+백업), 변호사 자문 완료.

## Out of Scope

- 기존 006 정치해설 모드의 UI·로직 변경
- 완전 자동 업로드 (운영자 최종 확인 단계는 필수 유지)
- 네이버·카카오TV 등 타 플랫폼 동시 배포 자동화
- 해외 시청자 대상 다국어 자막·음성
- YouTube 멤버십·슈퍼챗·Patreon 등 대체 수익 모델 통합
- 공격형·자극형 콘텐츠 템플릿 — perspective 무관 차단 (원칙 IV)
- BGM 라이선스 계약 대행
- 개인 신상 노출 방지 부가 기능 (익명 채널 운영은 운영자 책임)
- 세 번째 이상 perspective 축 추가 (현재 `dem`/`ppp`만 지원, 무소속·제3당 전용 모드 별도 계획 시 신규 feature로 분리)
- 단일 채널 내 두 perspective 혼합 운영 (perspective ↔ channel_id 1:1 하드블록)
