> **작성일**: 2026-04-17
> **작성 모드**: C 옵션 (A 일반화 패턴 + B 측정 프로토콜 병행)
> **모듈 매핑 방식**: ② CLAUDE.md 기반 모듈 레벨 매핑 (함수 시그니처 단정 회피)
> **편집 방침**: 정치적 입장 무관, 제작 기법 위주, 사실 기반 편집 기술만 다룸

## 0. 본 문서의 한계 및 사용 방법 (먼저 읽어주세요)

본 기획안은 두 부분으로 구성됩니다.

| 섹션 | 신뢰 수준 | 사용 방법 |
|---|---|---|
| **A. 일반화된 패턴 분석** (§1~§7) | "한국 정치 유튜브 카테고리 전반에서 널리 관찰되는 공통 기법" 수준의 일반화. 채널별 시그니처는 단정하지 않음. | 즉시 ContentsMaker 개선 의사결정에 사용 가능 |
| **B. 측정 프로토콜** (§9) | 사용자 또는 별도 리서처가 채널별 영상을 실제 시청·측정하기 위한 체크리스트. | 채널별 정밀 비교가 필요할 때 직접 측정 후 본 문서 §1~§7에 보강 |
| **ContentsMaker 매핑** (§8) | CLAUDE.md 기재 모듈 구조 기반. 모듈 레벨까지만 매핑. 함수 시그니처는 실제 코드 확인 후 `/plan` 단계에서 확정 권장. | 모듈 단위 작업 범위 산정에 사용 |

**다루지 않는 것**:
- 특정 채널의 정치적 입장 평가
- 클릭베이트·허위 정보 제작 기법
- 측정 없는 수치 단정 ("○○ 채널은 ○○색을 70% 사용한다" 등)

## 1. 후킹 (첫 3초) — 일반화 패턴

### 1.1 롱폼(8~15분) 인트로 공통 패턴

한국 정치 롱폼은 첫 3~5초에서 다음 중 1~2개를 결합하는 경우가 일반적입니다.

| 기법 | 설명 | 빈도 |
|---|---|---|
| **콜드 오픈 (Cold Open)** | 인트로 로고 없이 본론 발언 클립 즉시 재생 | 시사·뉴스 계열에서 흔함 |
| **티저 자막** | "오늘 ○○에서 무슨 일이 있었나" 같은 한 줄 카피를 2~3초 큰 자막으로 표시 | 매우 흔함 |
| **하이라이트 컴필레이션** | 본 영상에서 가장 자극적인 발언 3~5컷을 5~10초로 빠르게 편집 | 트렌드성 패턴 |
| **호스트 직접 인트로** | 진행자가 카메라 정면을 보며 "오늘 다룰 주제는…" | 정통 시사 채널 |
| **BGM 빌드업** | 저음 드론(drone) → 임팩트 사운드와 함께 자막 등장 | 트레일러 스타일 |

### 1.2 쇼츠(30~60초) 후킹 공통 패턴

쇼츠는 첫 1.5초가 결정적이며 다음 패턴이 알고리즘상 유리한 것으로 업계에서 통용됩니다.

| 기법 | 설명 |
|---|---|
| **선언형 후킹 자막** | "이거 진짜 충격입니다", "잠깐만요", "이 발언 들어보세요" 등 큰 자막 + 0.3초 zoom-in |
| **발언 즉시 재생** | 정치인 발언의 가장 강한 부분을 0초부터 재생, 인트로 컷 생략 |
| **질문형 후킹** | "○○ 의원이 왜 이런 말을 했을까요?" 형태의 호기심 유발 |
| **숫자 강조** | "3초만에", "1분 정리" 등 시간 약속 |
| **반응샷 우선** | 다른 의원/시민의 놀란 표정을 0~1초에 먼저 보여주고 발언 컷으로 전환 |

### 1.3 후킹 실패의 공통 원인

- 채널 로고/오프닝 시퀀스가 3초 이상 → 이탈률 급증
- 호스트가 인사말로 시작 ("안녕하세요 여러분")
- 본론 발언이 5초 후 등장

## 2. 자막 디자인 — 일반화 패턴

### 2.1 폰트 일반 경향

한국 정치 유튜브에서 **널리 사용되는 폰트군** (특정 채널 단정 X):

| 폰트군 | 특징 | 일반적 용도 |
|---|---|---|
| **본고딕 / Noto Sans KR Black** | 라이선스 자유, 두꺼움, 가독성 높음 | 메인 자막 가장 흔함 |
| **배달의민족 한나체 / 도현체** | 손글씨/캐주얼 느낌 | 코미디·풍자 톤 |
| **Spoqa Han Sans** | 모던, 깔끔 | 분석·해설 톤 |
| **G마켓 산스 / 에스코어드림** | 개성있는 굵은 폰트 | 후킹 자막, 강조 컷 |
| **나눔손글씨류** | 손글씨 느낌 | 시민 인터뷰, 댓글 인용 |

**라이선스 주의**: 상업용 가능 여부는 폰트별로 다름. ContentsMaker는 현재 Remotion 기본 폰트(시스템 폰트)를 사용하는 것으로 추정되므로, 본고딕(SIL OFL) 또는 Noto Sans KR(SIL OFL)이 가장 안전한 기본값입니다.

### 2.2 자막 시각 디자인 공통 요소

```
[일반적인 정치 쇼츠 자막 스펙 — 업계 통용 수준]

크기:        세로 9:16 영상 기준 자막 글자 높이 = 영상 높이의 6~9%
                (1080×1920 영상에서 110~170px)
굵기:        Black / ExtraBold (700~900)
색상:        흰색 본문 + 빨강/노랑 키워드 강조
외곽선:      검정 4~8px (가독성 핵심)
그림자:      검정 drop shadow blur 4px, opacity 60%
배경박스:    어두운 반투명 박스(opacity 70%) 또는 외곽선만
줄간격:      1.2~1.4
한 줄 글자수: 9~13자 (쇼츠) / 18~22자 (롱폼)
표시 시간:   최소 1.2초 (한국어 평균 읽기 속도)
```

### 2.3 키워드 강조 패턴

- **빨강 (#FF2D2D 계열)**: 비판·분노·고발 키워드 ("위반", "거짓말", "충격")
- **노랑 (#FFD700 계열)**: 핵심 정보·인물명·금액·날짜
- **하늘색/연두**: 인용·해설·설명 보조
- **하이라이트 박스**: 키워드 뒤에 색상 박스 깔기 (형광펜 효과)

### 2.4 화자 자막 vs 해설 자막 구분

업계에서 일반적으로 사용되는 시각적 구분:

| 구분 | 위치 | 디자인 |
|---|---|---|
| **화자 발언 자막** (원본) | 화면 하단 (영상 높이 75~88%) | 화자 이름 라벨 + 발언 인용 |
| **해설/내레이션 자막** | 화면 상단 또는 중앙 (영상 높이 15~30%) | 큰 글씨, 키워드 강조, 이모지 가능 |
| **사실 체크/캡션** | 화면 좌상단 | 작은 흰 글씨 + 회색 배경 |

## 3. 구성 / 편집 패턴

### 3.1 컷 길이 일반 경향

| 영상 유형 | 일반적 평균 컷 길이 | 특징 |
|---|---|---|
| **쇼츠 30~60초** | 1.5~2.5초 | 빠른 전환, 정지화면 즉시 컷 아웃 |
| **롱폼 분석형** | 4~8초 | 호스트 컷 + 자료 컷 교차 |
| **롱폼 토크쇼** | 6~12초 | 화자별 컷 |
| **하이라이트 영상** | 3~5초 | 발언별로 끊어 편집 |

**주의**: "1.5초 컷"은 업계 통용 감각 수치이며, 실제 채널별 측정은 §9.3 프로토콜로 확인 필요합니다.

### 3.2 핵심 발언 처리 패턴

발언이 임팩트 있을 때 흔히 사용되는 편집:

1. **반복 재생 (Replay)**: 같은 발언을 2~3회 반복, 두 번째는 줌인, 세 번째는 슬로우
2. **슬로우 모션**: 0.5x~0.7x로 핵심 단어 강조 (예: "그게 사실입니까?")
3. **정지 + 자막**: 화면 정지 + 큰 자막 + 효과음 (1~1.5초)
4. **반응샷 인서트**: 다른 의원/시민의 표정 컷 0.5~1초 삽입
5. **원본 + 해설 자막 동시 표시**: 화자는 말하고, 화면 상단에 해설자막

### 3.3 줌·패닝 사용 패턴

- **켄 번스 효과 (Ken Burns)**: 정지 사진/문서를 천천히 줌인+패닝
- **펀치 줌 (Punch Zoom)**: 키워드 발언 순간 0.2초간 빠르게 1.0x → 1.15x
- **호스트 정면 컷**: 카메라 정면, 줌 없음, 5~8초 유지
- **자료화면**: 뉴스 캡처/문서를 9:16에 맞게 패닝

### 3.4 인서트 클립 (B-roll) 활용

| 인서트 종류 | 일반적 용도 |
|---|---|
| **국회 본회의장 와이드샷** | 발언 배경 설명 |
| **정치인 표정 클로즈업** | 반응샷 |
| **자료 화면 / 그래프** | 통계·금액·일자 인용 |
| **뉴스 헤드라인 캡처** | 사실 확인 근거 |
| **시민 인터뷰** | 여론 반영 |
| **밈/짤방 (코미디 채널)** | 유머 강조 — 단, 저작권 주의 |

## 4. 사운드 디자인

### 4.1 BGM 일반 경향

| 카테고리 | 톤 | 사용 장면 |
|---|---|---|
| **트레일러 / 시네마틱 드론** | 저음 긴장감 | 후킹 인트로, 폭로 도입부 |
| **뉴스 BGM (피아노 + 스트링)** | 진지·차분 | 분석·해설 본론 |
| **빌드업 + 임팩트** | 점진적 상승 + 큰 타격음 | 핵심 발언 직전 |
| **코미디 BGM** | 우쿨렐레, 호기심 BGM | 풍자·코믹 채널 |
| **무음 / 환경음** | 원본 발언 강조 | 임팩트 컷 |

**볼륨 가이드** (일반 통용):
- BGM: -20~-25 dB (원본 음성 대비)
- 원본 발언: -3~-6 dB (피크)
- 더빙 해설: 0 dB ~ -3 dB
- 효과음: -10~-15 dB

### 4.2 효과음 (SFX) 사용 패턴

| 효과음 | 사용 시점 |
|---|---|
| **Whoosh (휙)** | 컷 전환, 자막 등장 |
| **Impact / Boom** | 핵심 발언 강조, 정지 컷 |
| **Ding / Bell** | 새 정보 등장, 사실 체크 |
| **Glitch / Static** | 의외성·놀람 |
| **Typewriter** | 자막 한 글자씩 등장 |
| **Riser** | 빌드업 (긴장감 상승) |
| **Page turn** | 챕터 전환 |
| **News sting** | 뉴스 헤드라인 인용 |

**무료 SFX 라이브러리 (상업용 가능)**:
- Freesound.org (CC0/CC-BY 필터링 필수)
- YouTube Audio Library
- Pixabay Sound Effects
- Zapsplat (가입 무료)

### 4.3 음성 처리

| 처리 | 용도 |
|---|---|
| **EQ (저역 컷 80Hz)** | 마이크 잡음 제거 |
| **컴프레서** | 다이내믹 평탄화 |
| **에코/리버브** | 강조 컷 (과도 사용 시 신뢰도 하락) |
| **피치 다운** | 코믹·풍자 |
| **스피드 1.05~1.1x** | 쇼츠 압축 (현재 ContentsMaker도 +20% rate 사용 중) |

## 5. 썸네일 / 제목 패턴

### 5.1 썸네일 시각 패턴 (롱폼)

업계 일반 패턴:

| 요소 | 빈도 / 디자인 |
|---|---|
| **인물 표정 클로즈업** | 매우 흔함 — 놀람·분노·황당 표정 |
| **빨강 + 노랑 대비** | 빨강 배경 + 노랑 텍스트 또는 그 반대 |
| **굵은 텍스트 (12~20자)** | 영상 면적의 30~40% 차지 |
| **화살표 / 동그라미** | 인물·문서 강조 |
| **숫자 강조** | "3개", "5초" 등 수치 키워드 |
| **vs 구도** | 두 인물 대비 (코미디·대결 구도) |

**주의**: 썸네일은 사실 기반이어야 함. 영상 내용과 무관한 자극적 이미지는 YouTube 정책 위반 가능.

### 5.2 제목 양식 (사실 기반)

권장 양식 (클릭베이트 회피):

```
[좋은 예]
"○○ 의원, 본회의장에서 ○○ 발언 — 1분 정리"
"○○ 법안 핵심 쟁점 3가지"
"오늘 국회 화제 발언 TOP 3"

[피해야 할 예 — 클릭베이트]
"충격! 이걸 진짜 했다고?"
"믿을 수 없는 ○○의 정체"
"이거 보면 잠 못 잡니다"
```

**제목 패턴 분석**:
- 길이: 25~45자 (모바일에서 잘리지 않는 범위)
- 인물명/법안명 등 검색 키워드 포함
- 대괄호 [속보], [팩트체크], [정리] 등 카테고리 라벨
- 숫자 사용 (3가지, 1분, TOP 5)

## 6. 톤 & 메시지

### 6.1 해설 화법 카테고리

| 톤 카테고리 | 특징 | 채널 유형 |
|---|---|---|
| **차분한 분석** | 중립적 어조, 사실 나열, 데이터 중심 | 정통 시사·해설 |
| **분노·고발** | 격앙된 어조, 감정 강조 | 정치 비평 |
| **풍자·비꼼** | 코미디 톤, 과장된 리액션 | 코미디 정치 |
| **친근·해설** | 친구처럼 설명, 반말/존댓말 혼용 | MZ 타깃 |
| **다큐멘터리** | 차분한 내레이션, 자료 중심 | 분석형 롱폼 |

**ContentsMaker 현재 톤**: edge-tts `ko-KR-SunHiNeural` + `+20%` rate → **친근·해설 톤**에 가까움. 다른 톤 확장 시 voice 또는 rate 조정 필요.

### 6.2 콜 투 액션 (CTA) 패턴

영상 종료부 1~5초:

- "구독·좋아요 부탁드립니다" (가장 일반)
- "댓글로 의견 남겨주세요" (참여 유도)
- "다음 영상에서 만나요" (이탈률 ↓)
- "알림 설정 잊지 마세요"
- 쇼츠는 CTA 짧게 (0~2초) 또는 생략하고 바로 다음 영상으로 흐름 연결

## 7. 알고리즘 최적화

### 7.1 영상 길이

| 형식 | 일반적 최적 구간 |
|---|---|
| **쇼츠** | 28~58초 (60초 직전까지 길게 유지하면 시청 유지율 ↑) |

### 9.1 측정 대상 채널 선정

[진보계]      매일경제 김용민TV / 새가는방송
[MZ 타깃]     쥐픽쳐스 / 빠더너스 (정치 콘텐츠 한정)
[쇼츠 특화]   각 채널의 쇼츠 5개씩 별도 측정

### 9.3 평균 컷 길이 측정법

```

  컷이 바뀔 때마다 카운트

  $ ffmpeg -i video.mp4 -filter:v "select='gt(scene,0.3)',showinfo" -f null -
  출력에서 scene change 타임스탬프 카운트

방법 3 (자동 — 본 프로젝트 활용):
  src/dem_shorts/editor/segment_cutter.py 의 컷 검출 로직을
  외부 영상에 적용 (별도 스크립트 작성 필요)
```

### 9.5 측정 시 주의사항

- **저작권 존중**: 측정용 다운로드 X. 스트리밍 시청 후 기록만.
- **편향 회피**: 채널을 카테고리별로 골고루 (보수/중도/진보 각 2개 이상)
- **측정자 편향**: 가능하면 2명 이상이 같은 영상을 측정해 평균
- **시기 편향**: 같은 시기 영상으로 통일 (예: 모두 최근 30일 내)
- **측정 자체가 시청 데이터에 영향**: 측정용 별도 계정 권장

## 10. 종합 권고 (Conclusion First)

**결론**:
1. **ContentsMaker는 파이프라인 자동화는 업계 평균 이상**이다. 부족한 것은 "편집 디테일 + 후킹 + 시각 디자인 + 사운드 매칭" 영역이다.
2. **즉시 ROI가 가장 높은 작업은 QW-01 (후킹 자막) + QW-03 (자막 외곽선) + QW-08 (클릭베이트 가드)**. 코드 변경 최소, 시각·신뢰도 영향 최대.
3. **중기 핵심은 BGM/SFX 자동 매칭 (MID-01) + 썸네일 자동 생성 (MID-05)**. 둘 다 채널 노출과 직결.
4. **장기 R&D는 시청 유지율 학습 루프 (LT-03)**. YouTube Analytics 데이터를 다음 영상 편집에 피드백하면 채널 성장 속도가 단계적으로 빨라진다.
5. **편향 없는 제작 기법만 도입하라**. "분노 톤 BGM"은 가능하나 "특정 정치인 비방 자동 생성" 같은 LLM 프롬프트는 금지. `claude_analyzer.py` 프롬프트 가드에 명시 필요.
